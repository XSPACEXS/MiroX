# Startup Board Build Process

## Introduction

Building startup boards is fundamentally iterative. Unlike a corporate dashboard that is built once and refreshed, startup boards are living documents that evolve as the business model evolves. The build process must account for rapid change, hypothesis invalidation, and the need to maintain both internal (honest) and external (polished) versions of the story. This guide covers the complete workflow from initial discovery through ongoing maintenance.

---

## Phase 1: Discovery — Understanding the Startup Context

### Duration: 30-60 minutes

### Key Questions

Before placing a single element on the board, answer these questions:

**Business Context**:

- What stage is the startup? (Idea, pre-seed, seed, Series A, growth)
- What is the primary use case for this board? (Internal strategy, investor presentation, team alignment, accelerator application)
- How many pivots or iterations has the startup gone through?
- What is the current funding status? (Bootstrapped, pre-seed raised, raising now)

**Audience**:

- Who is the primary audience? (Founders only, investors, team, advisors, board members)
- What is the audience's startup literacy? (First-time founders need more structure; serial entrepreneurs need flexibility)
- Will this board be presented live or shared asynchronously?

**Content Readiness**:

- Has the team done customer discovery? How many interviews?
- Are there validated vs. hypothesized elements?
- Do they have metrics or is this pre-product?
- Have they created a pitch deck, business plan, or previous canvas?

**Board Type Selection**:

| Primary Need                     | Board Type             | Complexity |
| -------------------------------- | ---------------------- | ---------- |
| Define/refine business model     | Lean Canvas            | Medium     |
| Prepare for fundraising          | Pitch Deck Storyboard  | Medium     |
| Manage investor conversations    | Investor Pipeline      | Advanced   |
| Scope the minimum viable product | MVP Planning Canvas    | Medium     |
| All of the above (startup hub)   | Multi-board linked set | Advanced   |

### Discovery Output

A brief document (can be notes on the board itself) capturing:

- Startup stage and context
- Board type selected
- Audience and use case
- Content readiness assessment (what exists vs. what needs to be created)
- Known hypotheses and their current validation status

## Phase 2: Structure — Defining the Information Architecture

### Duration: 20-40 minutes

### Architecture by Board Type

#### Lean Canvas Structure

```
Level 1: Canvas Header (identity + version)
Level 2: Nine blocks arranged in canonical Lean Canvas grid
  - Problem | Solution | UVP | Unfair Advantage | Customer Segments
  - Key Metrics (under Solution) | Channels (under UVP)
  - Cost Structure | Revenue Streams
Level 3: Within each block: hypotheses, evidence, sub-sections
Level 4: Hypothesis Validation Tracker (below canvas)
```

#### Pitch Deck Storyboard Structure

```
Level 1: Narrative Arc Guide (the story structure)
Level 2: Slide Cards (10 cards, 2 rows of 5)
  - Each card: Content, Visual Direction, Talking Points
Level 3: Preparation Notes (Q&A, demo flow, rehearsal log)
```

#### Investor Pipeline Structure

```
Level 1: Fundraise Overview (round, target, committed, pipeline)
Level 2: Pipeline Columns (Research → Outreach → Meeting → DD → Term Sheet)
  - Each column: Investor Cards with status and notes
Level 3: Term Sheet Comparison + Milestone Planning
```

#### MVP Planning Canvas Structure

```
Level 1: MVP Header (product, user, problem, goal, constraints)
Level 2: Planning Panels
  - Feature Backlog | Prioritization Matrix | MVP Scope
Level 3: Execution Panels
  - Build Timeline | Validation Experiments
```

### Zone Mapping

For each board, identify the three information zones:

| Zone       | Purpose                      | Startup Application                         |
| ---------- | ---------------------------- | ------------------------------------------- |
| Glanceable | Instant status at 25% zoom   | Board title, version, validation badges     |
| Scannable  | Key details at 50% zoom      | Block headers, key metrics, pipeline counts |
| Readable   | Full content at 75-100% zoom | Hypotheses, evidence, investor notes        |

## Phase 3: Skeleton — Building the Visual Framework

### Duration: 30-60 minutes

### Step-by-Step Skeleton Build

#### Lean Canvas Skeleton

1. **Set board size**: 5000 x 3500px
2. **Place header frame**: Full width, 320px height, positioned at (0, 0)
3. **Build the 3x3 grid**:
   - Row 1: Problem (900x1000), Solution (900x500, top), UVP (900x1000), Unfair Advantage (700x500, top), Customer Segments (900x1000)
   - Row 1 sub: Key Metrics (900x500, under Solution), Channels (700x500, under Unfair Advantage)
   - Row 2: Cost Structure (2400x550), Revenue Streams (2600x550)
4. **Place validation tracker**: Full width, 450px height, below revenue row
5. **Add block headers**: UPPERCASE, 28px bold, centered in each block
6. **Add block borders**: 2px #E0E0E0 between all blocks

#### Pitch Deck Skeleton

1. **Set board size**: 6000 x 3800px
2. **Place narrative arc guide**: Full width, 200px height
3. **Place 10 slide cards**: 1100x1400px each, 30px gaps, 5 per row
4. **Place preparation strip**: Full width, 600px height, below second row
5. **Add slide numbers and titles**: "SLIDE 1: HOOK", "SLIDE 2: PROBLEM"
6. **Add internal dividers**: Lines within each card separating Content / Visual / Talking Points

#### Investor Pipeline Skeleton

1. **Set board size**: 5600 x 3200px
2. **Place fundraise header**: Full width, 300px height
3. **Place 6 pipeline columns**: 850x1600px each, 20px gaps
4. **Place bottom panels**: Term Sheet Comparison (2600x800) + Milestone Planning (2800x800)
5. **Add column headers**: Stage names with color backgrounds
6. **Leave space for investor cards**: 280x160px cards with 15px vertical gaps

#### MVP Planning Skeleton

1. **Set board size**: 5600 x 3200px
2. **Place MVP header**: Full width, 280px height
3. **Place top row**: Backlog (1600x1400) + Matrix (1800x1400) + Scope (1800x1400), 30px gaps
4. **Place bottom row**: Timeline (2600x1000) + Experiments (2800x1000), 30px gaps
5. **Add section headers and internal structure**

### Skeleton Validation Checklist

Before adding content, verify:

- [ ] All sections are proportionally sized and aligned
- [ ] Headers are positioned and styled consistently
- [ ] Grid lines and borders are uniform
- [ ] Board size accommodates all planned sections
- [ ] Zoom at 25% shows the overall structure clearly
- [ ] Frame titles are visible at 50% zoom
- [ ] Internal padding (20px) is consistent across all blocks

## Phase 4: Content — Populating with Real Data

### Duration: 1-3 hours (varies dramatically by content readiness)

### Content Population Order

#### Lean Canvas Content Order

Fill blocks in this order (most foundational first):

1. **Problem** — The foundation. If this is wrong, everything else is wrong.
2. **Customer Segments** — Who has this problem?
3. **UVP** — What value do we deliver to that segment for that problem?
4. **Solution** — How do we deliver the UVP?
5. **Unfair Advantage** — Why can we win?
6. **Key Metrics** — How do we know it is working?
7. **Cost Structure** — What does it cost to operate?
8. **Revenue Streams** — How do we make money?
9. **Validation Tracker** — What is the current state of our learning?

This order follows the logical dependency chain. Problem and Segments define the opportunity. UVP and Solution define the response. Advantage and Metrics define the strategy. Costs and Revenue define the economics.

#### Pitch Deck Content Order

1. **Slide 1 (Hook)** and **Slide 10 (The Ask)** — Bookend the narrative first
2. **Slide 2 (Problem)** — Import from Lean Canvas
3. **Slide 3 (Solution)** — Import from Lean Canvas
4. **Slide 5 (Traction)** — Current metrics and proof points
5. **Slide 6 (Market)** — TAM/SAM/SOM sizing
6. **Slide 7 (Business Model)** — Revenue model and pricing
7. **Slide 8 (Team)** — Founder credentials and key hires
8. **Slide 9 (Financials)** — Projections and unit economics
9. **Slide 4 (Why Now)** — Market timing and tailwinds

### Content Quality Gates

For each content block, verify:

- [ ] Hypothesis status badge is assigned (VALIDATED / TESTING / HYPOTHESIS / INVALIDATED / PARTIAL)
- [ ] Evidence is cited for all validated claims (method, sample, date, result)
- [ ] Numbers are precise, not rounded (use $1,363, not ~$1.4K)
- [ ] Honest qualifiers are present ("We believe..." for hypotheses, "Evidence shows..." for validated)
- [ ] Content is written from the customer's perspective (problems) or founder's perspective (solutions)
- [ ] Cross-references to other boards are included where relevant

## Phase 5: Visual Polish — Elevating to Presentation Quality

### Duration: 30-60 minutes

### Polish Checklist

**Color and Consistency**:

- [ ] All validation badges use the standard color system (green/yellow/blue/gray/red)
- [ ] Sticky notes within each canvas block use the designated color
- [ ] Header bars use primary color background with white text
- [ ] All text meets contrast requirements

**Typography**:

- [ ] H1 (36px) used only for board title
- [ ] H2 (28px) used for all block headers, UPPERCASE
- [ ] Body text (14px) is consistent throughout
- [ ] KPI values (42px) stand out from surrounding text
- [ ] Version labels (12px) are present and positioned correctly

**Alignment and Spacing**:

- [ ] All blocks are pixel-aligned (no fractional positioning)
- [ ] Internal padding is 20px in all blocks
- [ ] Sticky notes within blocks have 12-15px gaps
- [ ] Canvas blocks have 0px gaps (borders delineate)
- [ ] Non-canvas sections have 30-40px gaps

**Presentation Mode** (for investor-facing boards):

- [ ] Frames are configured for Miro presentation mode
- [ ] Each frame fills the screen at presentation resolution
- [ ] Frame order follows the narrative arc
- [ ] Frames have clean titles that serve as slide titles

## Phase 6: Review — Quality Assurance

### Duration: 20-30 minutes

### The Founder Review

Walk the board as if you are the founder presenting to an investor:

1. **5-second test**: At 25% zoom, can you immediately identify the board type, company name, and current version?
2. **30-second scan**: At 50% zoom, can you identify the key metrics, validation status, and overall business model?
3. **2-minute read**: At 75% zoom, does the content tell a coherent story from problem to opportunity to plan?
4. **Investor question test**: Can the board answer these without additional context?
   - What problem are you solving? (Problem block)
   - Who is your customer? (Customer Segments)
   - What makes you different? (UVP + Unfair Advantage)
   - Is it working? (Key Metrics + Validation Tracker)
   - How much are you raising and why? (Revenue + Ask)

### The Honesty Review

Verify intellectual honesty:

- [ ] No hypothesis is presented as a fact
- [ ] At least one INVALIDATED or failed experiment is documented (if applicable)
- [ ] The Unfair Advantage block includes honest limitations
- [ ] Financial projections are labeled as targets or projections, not actuals
- [ ] Confidence levels are stated for key assumptions

### The Consistency Review

Check cross-board consistency:

- [ ] Numbers on the Lean Canvas match the Pitch Deck
- [ ] Customer segment on the Canvas matches the Pitch Deck slide
- [ ] Revenue model on the Canvas matches the Financial slides
- [ ] Version numbers are updated across all linked boards

## Phase 7: Maintenance — Keeping the Board Alive

### Update Cadence

| Board Type         | Update Frequency | Trigger for Update                                     |
| ------------------ | ---------------- | ------------------------------------------------------ |
| Lean Canvas        | Weekly           | New experiment result, pivot decision, metric change   |
| Pitch Deck         | Per fundraise    | Before each investor meeting or batch of meetings      |
| Investor Pipeline  | Daily (active)   | After every investor interaction                       |
| MVP Planning       | Per sprint       | Sprint planning, feature completion, experiment result |
| Validation Tracker | Per experiment   | Experiment completion, hypothesis status change        |

### Version Control Protocol

Every significant update should:

1. **Increment the version number**: v2.3 → v2.4 (minor update) or v2.3 → v3.0 (pivot)
2. **Update the date**: "Updated March 15, 2024"
3. **Log the change**: Brief note in the version history: "v2.4: Invalidated H4, updated pricing hypothesis"
4. **Archive the previous version**: Either as a linked board or a captured screenshot

### Iteration Logging

Maintain a version history section on each board:

```
VERSION HISTORY:
v3.0 (Mar 15) — Pivoted from SMB to mid-market after H2 invalidated
v2.4 (Mar 8)  — Updated pricing: $29 Starter validated, testing $99 Pro
v2.3 (Mar 1)  — Added 47-customer traction data, NPS tracking
v2.2 (Feb 22) — H1 validated (12+ hrs/week manual reporting confirmed)
v2.1 (Feb 15) — MVP launched to first 20 beta users
v2.0 (Feb 1)  — Major pivot: from "AI reporting" to "automated reconciliation"
v1.0 (Jan 10) — Initial canvas based on 10 customer interviews
```

## Quick-Start Sequences

### 30-Minute Lean Canvas

For a founder who needs a canvas TODAY:

1. (2 min) Create board, place header with startup name and today's date
2. (3 min) Draw the 3x3 grid using shapes (rectangles with borders)
3. (3 min) Label all 9 blocks with UPPERCASE headers
4. (5 min) Fill Problem block with top 3 problems (even if hypotheses)
5. (5 min) Fill Customer Segments with primary segment and early adopters
6. (5 min) Fill UVP with single sentence and high-level concept
7. (3 min) Fill Solution with top 3 features
8. (2 min) Mark all blocks with [HYPOTHESIS] badges
9. (2 min) Add version label: "v1.0 | [today's date] | Pre-validation"

### 45-Minute Pitch Deck Storyboard

1. (5 min) Create board, place narrative arc guide
2. (5 min) Create 10 slide card frames with numbers and titles
3. (5 min) Fill Slide 1 (Hook) with the strongest opening stat
4. (5 min) Fill Slide 2 (Problem) from Lean Canvas
5. (5 min) Fill Slide 3 (Solution) from Lean Canvas
6. (5 min) Fill Slide 10 (The Ask) with round size and use of funds
7. (5 min) Fill remaining slides with bullet points (content outlines)
8. (5 min) Add visual direction notes to each slide
9. (5 min) Review narrative flow and adjust sequence if needed

### 20-Minute Investor Pipeline Setup

1. (3 min) Create board, place fundraise header with round target
2. (5 min) Create 6 pipeline columns with stage headers
3. (10 min) Add top 10 target investors as cards in the Research column
4. (2 min) Add term sheet comparison placeholder and milestone panel

## Multi-Board Linking

### Creating a Startup Board Hub

When a startup needs multiple boards, create a hub board that links them:

```
STARTUP HUB — DataSight AI

+------------------+  +-------------------+
| LEAN CANVAS v2.3 |  | PITCH DECK v1.4   |
| [Link to board]  |  | [Link to board]   |
| Last updated:    |  | Last presented:   |
| March 15, 2024   |  | March 12, 2024    |
+------------------+  +-------------------+

+------------------+  +-------------------+
| INVESTOR PIPELINE|  | MVP PLAN v1.1     |
| [Link to board]  |  | [Link to board]   |
| Active: 12 VCs   |  | Sprint 4 of 6     |
| Committed: $500K |  | On track          |
+------------------+  +-------------------+
```

### Cross-Board Synchronization Rules

When updating one board, check these dependencies:

| If You Change...  | Also Update...                                  |
| ----------------- | ----------------------------------------------- |
| Problem statement | Pitch Deck Slide 2, MVP Problem block           |
| Customer segment  | Pitch Deck Slide 6, MVP Target User             |
| Revenue model     | Pitch Deck Slides 7+9, Lean Canvas Cost/Revenue |
| Traction metrics  | Pitch Deck Slide 5, Lean Canvas Key Metrics     |
| Fundraise amount  | Pitch Deck Slide 10, Investor Pipeline header   |
| MVP scope         | Lean Canvas Solution block, Pitch Deck Slide 3  |
| Hypothesis status | Validation Tracker, affected Canvas blocks      |
