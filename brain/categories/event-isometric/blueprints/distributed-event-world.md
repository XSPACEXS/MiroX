# Distributed Event World

## Overview

- **Purpose**: A multi-venue event map designed as floating islands connected by bridges, where each island represents a different event venue or city hosting simultaneous distributed events. This world-building approach visualizes a globally distributed event program — showing timezone overlaps, venue capacities, local programming, cross-venue livestreams, and how attendees at one location can interact with sessions at another. The floating island metaphor makes geographic distribution feel cohesive and connected rather than fragmented.
- **Best For**: Companies running multi-city event series, distributed conference organizers (like GitHub Universe or Google I/O Extended), global hackathon coordinators, remote-first companies planning all-hands across offices, franchise event networks.
- **Complexity**: Advanced
- **Board Size**: 5000 x 3500px

## Example Content

NovaTech Connect 2026 — a globally distributed tech event across 4 physical cities and 1 virtual hub, with 4,800 total attendees, 72 sessions, and a shared gamification leaderboard. Each city has unique local programming while sharing keynotes and cross-venue interaction through the Virtual Hub.

## Variations

1. **Hackathon World**: Each island is a hackathon venue with shared challenges and a global scoreboard
2. **Sales Kickoff World**: Each island is a regional sales office with shared training and local celebrations
3. **Community Meetup Network**: Each island is a city meetup chapter with shared speakers via livestream
4. **Academic Conference Network**: Each island is a university hosting satellite sessions

## Build Instructions

1. Create board at 5000x3500px with Deep Space background
2. Place title banner at top
3. Arrange SF and London islands in the upper half, Tokyo and Sao Paulo in the lower half
4. Place Virtual Hub centrally connecting all islands
5. Draw gold bridge connectors between all islands and the Virtual Hub
6. Add dashed timezone bridges between adjacent cities
7. Place Timezone Sync Bar and Global Leaderboard at the bottom

## Tips & Best Practices

- Use Miro's presentation mode to "fly" between islands during cross-venue moments
- Embed countdown timers for timezone-sensitive shared sessions
- Link each island to a detailed sub-board with full local programming
- Update the leaderboard in real-time during the event (or mock it for planning)
- Color the bridges green/yellow/red based on connection quality during live events
- Archive this board as the definitive event record with post-event metrics overlay
