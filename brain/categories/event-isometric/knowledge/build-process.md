# Build Process — Event Guide Immersive Isometric

## Advanced Build Workflow

Building an immersive isometric event guide is the most visually demanding task in the MiRoX Brain. It requires balancing artistic world-building with practical information architecture. This guide covers the complete workflow from concept to live event companion.

---

## Phase 1: Event Information Gathering

**Time: 30-60 minutes**

### Step 1: Collect Event Information

- Event name, dates, location, expected attendance
- Complete session schedule with speakers, times, and rooms
- Venue map or floor plan (even a rough sketch helps)
- Sponsor list and booth assignments
- Logistical details: WiFi, food, parking, emergency info
- Speaker photos and bios (if available)

### Step 2: Define the Zone Architecture

- List all distinct areas of the event (main stage, breakout rooms, expo, networking, food, registration)
- Assign a unique color to each zone from the palette
- Rank zones by importance (anchor, primary, secondary, utility)
- Determine which zones need full isometric treatment vs flat reference treatment

### Step 3: Choose the Layout Pattern

- Isometric Conference Campus (standard, most events)
- Floating Islands (distributed/virtual events)
- Split-World (hybrid events)
- Festival Campus (outdoor/multi-day events)

### Step 4: Plan the Decorative Theme

- Daytime/bright (standard conferences) or nighttime/dark (galas, tech events)
- Outdoor (parks, festivals) or indoor (convention centers)
- Formal (corporate events) or casual (hackathons, festivals)
- This determines the decorative element choices and background treatment

---

## Phase 2: Board Foundation

**Time: 20-30 minutes**

### Step 5: Create the Board

- Set canvas size: 5000 x 3500px (standard) or larger for complex events
- Set background color based on theme:
  - Standard conference: Soft Cream (#FDFEFE)
  - Night/tech: Midnight Blue (#0C1445)
  - Outdoor/festival: Park Green (#E8F5E9)
- Enable grid alignment

### Step 6: Place the Title Banner

- Create full-width frame (4800 x 400px) at the top
- Background: Deep Slate (#2C3E50) or primary color
- Add event name (48-64pt, bold, white)
- Add dates, location, tagline (22-28pt, gold or accent)
- Add stats bar: attendees, sessions, speakers, exhibitors, days
- Add decorative elements in the banner (small shapes, geometric accents)

### Step 7: Create Zone Placeholder Frames

- Place empty frames at planned positions for each zone
- Set each to zone color at 8-10% opacity
- Label with zone title (28-36pt, zone color)
- Verify the layout creates a believable spatial arrangement
- Ensure there is space between zones for pathways and decorations

---

## Phase 3: Isometric World Construction

**Time: 1-2 hours**

### Step 8: Build the Ground Plane

- If using a ground treatment (outdoor events), create a large light-colored rectangle beneath all zones
- Add subtle texture or color variation to suggest terrain
- Position this behind all zone frames (lower z-order)

### Step 9: Build Zone Structures

For each zone, apply the isometric building treatment:

**Simple zones (utility, small areas):**

- Standard Miro frame with zone-colored background
- Clear header text
- Content organized within the frame

**Feature zones (main stage, expo, breakout):**

- Apply the three-face method:
  1. Main rectangle (front face) with zone color fill
  2. Top rectangle (roof) with lighter shade, offset for isometric effect
  3. Side rectangle (depth) with darker shade
- Add building details: entrance shapes, window shapes, flag/banner
- Add shadow beneath the building (dark gray at 15% opacity)

**Hero zones (the 1-2 most important areas):**

- Larger building with full three-face treatment
- Additional details: spotlights, stage elements, banners
- More elaborate entrance and decoration

### Step 10: Build the Pathway Network

- Draw main pathways (5px, golden yellow) connecting all major zones
- Create pathway labels at midpoints (walking time, direction)
- Draw secondary pathways (2-3px, light gray) for minor connections
- Ensure every zone is reachable via at least one pathway
- Add pathway decorations: trees every 200px, occasional human figures

---

## Phase 4: Zone Content Population

**Time: 30-60 minutes per zone**

### Step 11: Populate Each Zone

For each zone, add the appropriate content per the content guide:

**Registration zone:**

- Check-in desk labels, badge types, welcome kit info
- Hours and WiFi info
- Emergency contacts

**Main Stage zone:**

- Full session schedule with times, titles, speakers
- Capacity and AV setup notes
- Key session highlights

**Breakout Rooms zone:**

- Track descriptions and session lists per room
- Capacity per room
- Recording and Q&A tool info

**Expo Hall zone:**

- Sponsor tiers and booth assignments
- Demo theater schedule
- Special activities (scavenger hunt, prize draws)

**Networking zone:**

- Area descriptions and capacities
- Scheduled networking activities
- Tips and special features

**Food zone:**

- Meal schedule
- Dietary options
- Evening event details

**Workshop Labs zone:**

- Lab titles, durations, capacities
- Pre-requisites and registration requirements

### Step 12: Add the Schedule Reference

- Create the timeline agenda panel (flat, non-isometric)
- Position below the world or as a linked frame
- Color-code session blocks to match zone colors
- Add "Don't Miss" highlights

### Step 13: Add the Speaker Directory

- Create speaker cards for featured speakers
- Include: name, title, company, session, day/time, room
- Photo placeholders (will be replaced with actual photos when available)

### Step 14: Add the Companion Zone

- Create the real-time companion panel (800-1000px wide)
- Position at top-right or as a floating element
- Include: Now Playing, Up Next, Quick Links, Live Poll
- This is the ONLY zone that will change during the event

---

## Phase 5: Decorative World-Building

**Time: 30-60 minutes**

### Step 15: Add Sky Elements

- Place 3-5 cloud clusters in the sky area (top 30% of board)
- Vary cloud sizes and opacity (15-25%)
- For dark boards: add 15-20 stars and twinkles
- Position title banner within or below the sky area

### Step 16: Add Ground Elements

- Place trees along all pathways (alternating sides, every 200-300px)
- Add tree clusters in open spaces between zones
- Add bushes at zone entrances
- For outdoor events: add grass patches, flowers, benches

### Step 17: Add Human Figures

- Place 2-3 walking figures per main pathway
- Place 1-2 standing figures near zone entrances
- Place groups of figures near networking and community zones
- Place a speaker figure near the main stage
- Place seated figures in the food and lounge areas
- Total: 15-25 human figures across the board

### Step 18: Add Ambient Elements

- Music notes near stages
- WiFi signals near registration and lounges
- Speech bubbles near networking areas
- Spotlights above stages
- Coffee cups near food areas
- Banners/flags at zone entrances

### Step 19: Lock All Decorative Elements

- Select all decorative elements (clouds, trees, figures, ambient)
- Lock them (right-click > Lock)
- Verify none obstruct text or functional content
- Verify the locked elements cannot be accidentally moved

---

## Phase 6: Quality Review & Polish

**Time: 20-30 minutes**

### Step 20: The Four-Zoom Test

1. **Full board zoom-out**: Does it look like a beautiful isometric world? Are zones clear? Is the sky-ground relationship believable?
2. **Zone-level zoom**: Can you read zone titles? Are pathway labels visible? Can you identify which zones connect?
3. **Content zoom**: Is session information complete? Can you read times, titles, speakers? Are rooms labeled?
4. **Mobile zoom**: Can you tap a zone on a phone? Is the companion panel readable?

### Step 21: Content Accuracy Check

- Verify all session times against the official schedule
- Verify all speaker names and titles
- Verify all room names and capacities
- Verify WiFi password, emergency contacts, and logistics
- Verify sponsor logos and booth assignments

### Step 22: Presentation Mode Setup

- Create Miro frames for presentation mode
- Frame order: Title > Registration > Main Stage > Breakout > Expo > Networking > Food > Labs > Schedule > Speakers
- Each frame should capture one zone cleanly

---

## Phase 7: Event-Day Operations

### Step 23: Pre-Event Preparation

- Run through the entire board one final time
- Ensure the companion zone is ready to receive updates
- Brief the event coordinator on how to update the companion zone
- Test the board on mobile devices to ensure usability
- Share the board link via email, app, and printed QR codes at the venue

### Step 24: During the Event

- Update the companion zone every 15-30 minutes
- Update "Now Playing" and "Up Next" at each session change
- Post any schedule changes immediately
- Add capacity alerts if rooms are full
- Run live polls and display results

### Step 25: Post-Event Archiving

- Add session recording links within 48 hours
- Add photo gallery links within 1 week
- Update attendance actuals
- Add satisfaction survey results
- Lock the entire board and add "ARCHIVED — [Event Name] [Date]" watermark
- Duplicate the board structure (without content) as a template for the next event

## Time Estimates

| Phase                               | Estimated Time   |
| ----------------------------------- | ---------------- |
| Event Information Gathering         | 30-60 min        |
| Board Foundation                    | 20-30 min        |
| Isometric World Construction        | 1-2 hours        |
| Zone Content Population (all zones) | 4-8 hours        |
| Decorative World-Building           | 30-60 min        |
| Quality Review & Polish             | 20-30 min        |
| **Total**                           | **7 - 13 hours** |

Note: The first board takes longer. Subsequent events using the same template: 4-6 hours.

---

## Reusability Strategy

To make future event boards faster to build:

1. **Save the decorated world as a template** (zones, pathways, decorations — without content)
2. **Create a content input spreadsheet** that maps to zone fields (session title, speaker, time, room)
3. **Standardize zone sizes** so content from any event fits the same frame dimensions
4. **Keep the decorative elements as a separate locked layer** that never changes between events
5. **Document the color palette** so it can be reapplied instantly
