# Mind Maps — Visual Guide

## Overview

The visual treatment of a mind map determines whether it communicates instantly or confuses endlessly. This guide covers every visual decision — color, typography, element choice, contrast, decoration, and accessibility — with specific guidance for mind map boards. Every recommendation is grounded in a rationale: not just WHAT to do, but WHY it works.

## Typography Scale

### Heading Hierarchy

Mind maps use a compressed typography scale because content is distributed spatially rather than linearly. Unlike documents where headings create vertical hierarchy, mind maps encode hierarchy through both font size AND position.

| Level | Element          | Font Size | Weight          | Line Height | Usage                      |
| ----- | ---------------- | --------- | --------------- | ----------- | -------------------------- |
| 0     | Central Topic    | 28-36px   | Bold (700)      | 1.2         | The main question or topic |
| 0     | Central Subtitle | 16-20px   | Regular (400)   | 1.3         | Date, team, context        |
| 1     | Branch Heading   | 20-26px   | Semi-Bold (600) | 1.2         | Major branch labels        |
| 2     | Sub-Branch       | 16-20px   | Medium (500)    | 1.3         | Secondary branch labels    |
| 3     | Detail Item      | 13-16px   | Regular (400)   | 1.4         | Individual ideas, notes    |
| 4     | Annotation       | 10-13px   | Regular (400)   | 1.4         | Metadata, timestamps, tags |
| -     | Legend/Key       | 12-14px   | Medium (500)    | 1.3         | Color key explanations     |

### Font Recommendations

**Primary font**: Use a clean sans-serif that is highly legible at small sizes:

- Preferred: Inter, Roboto, Open Sans, Source Sans Pro
- Acceptable: Arial, Helvetica, Segoe UI
- Avoid: Decorative fonts, serif fonts (they lose legibility at small zoom levels), thin/ultralight weights

**Monospace font** (for technical content, code snippets, data):

- Preferred: JetBrains Mono, Fira Code, Source Code Pro
- Size: Same as the level, but use the mono variant

### Text Alignment Rules

- **Center node text**: Center-aligned, both horizontally and vertically
- **Branch headings**: Center-aligned within the branch node
- **Sticky note text**: Left-aligned (easier to scan when there are multiple lines)
- **Annotations**: Left-aligned, positioned below or beside the element they describe
- **Legend text**: Left-aligned with color swatches on the left

### Text Length Guidelines

- **Center node**: 5-15 words (a question works best: "How might we...?")
- **Branch heading**: 1-4 words (category labels: "Customer Experience")
- **Sub-branch**: 2-8 words (specific idea: "Reduce onboarding steps")
- **Detail item**: 3-15 words (actionable detail: "Simplify the signup form to 3 fields")
- **Annotation**: 5-20 words (context: "Based on Q2 survey — 67% of users cited this")

## Element Selection Guide

### When to Use Sticky Notes

Sticky notes are the workhorse of mind maps. Use them for:

- Individual ideas during brainstorming (one idea per sticky)
- Items that might be moved, reorganized, or voted on
- Content at Level 2 or Level 3 (sub-branches and details)
- When you want a "workshop" or "collaborative" feel

**Sizing**:

- Small sticky: 130x130px (for brief ideas, 1-3 words)
- Medium sticky: 200x200px (for standard ideas, 4-10 words)
- Large sticky: 270x270px (for detailed ideas, 10-20 words)

**Color coding**: Sticky note color should match the branch color (use a lighter tint, 30-50% opacity of the branch color, or use Miro's built-in sticky colors mapped to branches).

### When to Use Shapes

Shapes communicate permanence and structure. Use them for:

- The central topic node (rounded rectangle or circle)
- Branch heading nodes (rounded rectangles or ellipses)
- Decision points (diamonds)
- Status indicators (small circles: green/yellow/red)
- Emphasis containers (highlighting an important cluster)

**Shape guidelines**:

- Corner radius for rectangles: 8-16px (rounded but not circular)
- Stroke width: 2-4px for major shapes, 1-2px for minor shapes
- Fill: Solid color for headings, semi-transparent (80-90% opacity) for containers
- Avoid: Sharp rectangles (feel corporate and rigid), triangles (aggressive), irregular shapes (confusing)

### When to Use Text Blocks

Plain text blocks (without shape backgrounds) are for:

- Annotations and explanations
- Labels on connectors
- Metadata (dates, authors, versions)
- Legend entries
- Any text that should feel lightweight and non-intrusive

### When to Use Frames

Frames are invisible or subtle containers that group elements. Use them for:

- Enclosing an entire branch and its children
- Creating the cluster boundaries in Cluster Cloud layouts
- Defining zones (legend zone, action zone, meta zone)
- Grouping related sticky notes within a branch

**Frame guidelines**:

- Background: Transparent or very subtle tint (5-10% opacity of zone color)
- Border: 1-2px, dashed or dotted, light gray or branch color at 30% opacity
- Padding: 30-50px between frame border and nearest internal element
- Title: Optional — place at top-left of frame, 14-16px, medium weight

## Creating Contrast, Emphasis, and Focal Points

### The Three Levels of Emphasis

**Level 1 — Structural Emphasis (Built into the layout)**:

- Size: Larger elements draw attention first
- Position: Center and top elements are seen before periphery and bottom
- Color saturation: More saturated colors pop against muted ones
- These require no additional decoration — the layout itself creates emphasis

**Level 2 — Highlighted Emphasis (For key insights)**:

- Bold border: Increase stroke width to 4-6px on critical elements
- Background glow: Add a subtle drop shadow or outer glow (soft, 10-20px blur, branch color at 20-30% opacity)
- Star or badge: Place a small icon (star, lightning bolt, checkmark) next to breakthrough ideas
- Use sparingly — if more than 15% of elements have Level 2 emphasis, nothing stands out

**Level 3 — Call-to-Action Emphasis (For next steps)**:

- Distinct background color: Use the accent color (amber, orange, or gold) that contrasts with all branch colors
- Bold text with larger font size
- Arrow or pointer shape directing attention
- Reserve for 1-3 items per board maximum

### Contrast Techniques

**Color contrast**: Ensure a minimum 4.5:1 contrast ratio between text and background. This is not just accessibility — it is readability.

| Background Color | Recommended Text Color | Contrast Ratio |
| ---------------- | ---------------------- | -------------- |
| #1A237E (Navy)   | #FFFFFF (White)        | 12.6:1         |
| #F9A825 (Amber)  | #212121 (Charcoal)     | 10.8:1         |
| #00695C (Teal)   | #FFFFFF (White)        | 7.2:1          |
| #F5F5F0 (Pearl)  | #212121 (Charcoal)     | 15.3:1         |
| #121212 (Dark)   | #FFFFFF (White)        | 18.1:1         |

**Size contrast**: The center node should be at least 1.5x the size of branch nodes, which should be at least 1.3x the size of sub-branch nodes.

**Density contrast**: Areas with tightly packed items feel busy and active. Areas with space feel calm and resolved. Use this intentionally — dense clusters signal "lots of ideas here" while sparse clusters signal "this is settled."

## Icon and Emoji Usage

### When to Use Icons/Emojis

Icons and emojis serve as rapid visual identifiers. They should:

- Appear alongside branch headings to provide instant category recognition
- Mark special items (action items, decisions, risks, wins)
- Replace text labels where universal understanding exists

### Recommended Icon System for Mind Maps

| Purpose       | Recommended Icon/Emoji          | Usage                             |
| ------------- | ------------------------------- | --------------------------------- |
| Central topic | Target, lightbulb, brain        | Next to the main title            |
| Ideas         | Lightbulb, sparkle              | Marking creative ideas            |
| Questions     | Question mark, magnifying glass | Items needing investigation       |
| Decisions     | Checkmark, green circle         | Confirmed decisions               |
| Risks         | Warning triangle, red flag      | Risk items                        |
| Action items  | Arrow right, clipboard          | Tasks to execute                  |
| People        | Person silhouette, team         | Assignments or stakeholders       |
| Time          | Clock, calendar                 | Deadlines or time-sensitive items |
| Data          | Chart, graph                    | Items backed by data              |
| Love/Priority | Star, heart, fire               | Top-voted or high-priority ideas  |

### Icon Guidelines

- **Size**: 16-24px (should not dominate the text it accompanies)
- **Position**: Left of the text or above the text, consistently across the board
- **Color**: Match the branch color or use a neutral (black/white) for universal icons
- **Density**: Maximum 1 icon per element. More than that creates visual noise
- **Consistency**: If Branch A uses an emoji prefix, all branches should use emoji prefixes

## Decorative Elements

### Elements That Elevate

**Divider lines**: Thin horizontal or vertical lines (1-2px, light gray) that separate zones without enclosing them. Use between the mind map area and the legend, or between the content and the action items zone.

**Section badges**: Small rectangular badges (80-120px wide, 30-40px tall) with rounded corners and solid color fill, containing labels like "KEY INSIGHT," "ACTION REQUIRED," or "NEW." Place near the top-right corner of important items.

**Background texture**: A very subtle pattern (dots at 3-5% opacity, or a faint grid) behind the entire board. This adds visual warmth without distracting from content. Use sparingly and at very low opacity.

**Connector labels**: Small text labels placed at the midpoint of connector lines, explaining the relationship. Use a semi-transparent background behind the label text so it reads clearly over the connector.

**Progress indicators**: Small pie charts, progress bars, or numbered step indicators attached to items that have phases or completion states.

**Pull quotes**: For key insights or surprising findings, create a larger text block with a left border (4-6px, accent color) and italic text. This draws the eye to the most important takeaways.

### Elements to Avoid

- **Gradients on sticky notes**: They look dated and reduce readability
- **3D effects**: Drop shadows and bevels on individual stickies feel like 2005 PowerPoint
- **Clip art**: Low-quality illustrations undermine professionalism
- **Animated elements**: Miro supports some animation, but it distracts from static mind maps
- **Excessive borders**: Double borders, thick borders on everything, or borders on borders
- **Watermarks**: They clutter the canvas and rarely add value

## Dark Mode vs Light Mode

### Light Mode (Default)

- Background: #F5F5F0 to #FFFFFF
- Text: Dark (#212121 to #424242)
- Shapes: Medium-to-dark fills with white text, or light fills with dark text
- Connectors: Medium gray (#546E7A to #78909C)
- Best for: Printed boards, projected presentations, bright environments

### Dark Mode

- Background: #121212 to #1E1E1E
- Text: Light (#FFFFFF to #E0E0E0)
- Shapes: Bright, saturated fills with white text
- Connectors: Light gray (#90A4AE to #B0BEC5) or branch-colored at 60% opacity
- Best for: Screen-only viewing, extended work sessions, tech audiences

### Transition Rules

When converting a light board to dark (or vice versa):

1. Invert the background (light becomes dark)
2. Invert text colors (dark becomes light)
3. Increase saturation of shape fills by 15-25% (colors need to be stronger against dark backgrounds)
4. Lighten connector colors by 30-40%
5. Add a subtle outer glow (2-4px, white at 10%) around text blocks for legibility
6. Test all color combinations for contrast ratio compliance

## Accessibility Guidelines

### Color Blindness Safe Design

Approximately 8% of men and 0.5% of women have some form of color vision deficiency. Mind maps that rely solely on color to distinguish branches will fail for these users.

**Solutions**:

1. **Redundant coding**: Pair color with a second visual variable — shape, pattern, icon, or label
2. **Colorblind-safe palette**: Avoid red-green pairs as the only distinguishing factor. Use blue-orange, blue-yellow, or add texture/pattern differences
3. **Pattern fills**: Use subtle patterns (diagonal lines, dots, crosshatch) inside shapes for critical distinctions
4. **Labels always**: Every branch should have a text label in addition to its color

**Colorblind-safe branch palette** (works for protanopia, deuteranopia, and tritanopia):

| Branch | Color      | Hex     | Backup Pattern   |
| ------ | ---------- | ------- | ---------------- |
| 1      | Blue       | #1565C0 | Solid fill       |
| 2      | Orange     | #E65100 | Diagonal lines   |
| 3      | Light Blue | #039BE5 | Dotted fill      |
| 4      | Yellow     | #F9A825 | Horizontal lines |
| 5      | Dark Gray  | #37474F | Crosshatch       |
| 6      | Pink       | #AD1457 | Vertical lines   |

### Contrast Requirements

- **WCAG AA standard**: 4.5:1 contrast ratio for normal text, 3:1 for large text (18px+ or 14px+ bold)
- **WCAG AAA standard**: 7:1 for normal text, 4.5:1 for large text
- All mind map text should meet at minimum AA standard
- Center node and branch headings should meet AAA standard (they are critical navigation elements)

### Screen Reader Considerations

While Miro boards are inherently visual, consider adding:

- A text-based summary frame in the corner that lists all branches and key items linearly
- Alt-text descriptions on any embedded images
- Logical reading order: center, then branches clockwise starting from top
- Numbered branches (1, 2, 3...) to provide a clear sequence for non-visual navigation

### Motor Accessibility

For boards used in collaborative workshops:

- Minimum click target size: 44x44px (WCAG 2.1 requirement)
- Sticky notes should never be smaller than 100x100px
- Spacing between interactive elements: minimum 8px (prevents accidental selection)
- Provide keyboard navigation hints if the board will be used non-interactively

## Visual Quality Checklist

Before considering a mind map complete, verify:

- [ ] Center node is the most visually prominent element (largest, darkest/brightest)
- [ ] Each branch has a distinct color that is semantically meaningful
- [ ] Text is legible at the intended viewing zoom level
- [ ] Contrast ratios meet WCAG AA minimums for all text
- [ ] Connectors are clean, curved, and non-overlapping
- [ ] White space between branches creates clear separation
- [ ] Typography hierarchy is consistent across all levels
- [ ] Icons/emojis are used consistently (all branches have them, or none do)
- [ ] A legend or key exists if the board uses more than 4 colors
- [ ] The board works at three zoom levels: bird's eye, branch, and detail
- [ ] No element is orphaned (every item connects to the structure)
- [ ] Decorative elements add meaning, not just decoration
- [ ] The board has been tested against colorblind simulation tools
- [ ] Frame backgrounds are subtle (never compete with content for attention)
- [ ] The overall impression is "organized and inviting" not "cluttered and overwhelming"
