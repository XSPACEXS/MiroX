# Workshop Boards — Visual Guide

## Overview

Workshop boards use a functional visual language — clear, clean, and purposeful. Unlike
event boards (which embrace decorative richness) or design boards (which showcase
aesthetic craft), workshop boards strip away visual noise to maximize focus on the
facilitation process and participant contributions. Every visual choice on a workshop
board should answer: "Does this help participants do the workshop activity, or is it
just decoration?"

This guide provides the complete visual toolkit for workshop boards: color palettes,
typography scale, element selection, and accessibility requirements.

## Typography Scale

### Workshop Board Typography

Workshop boards use clear, functional typography. Readability is paramount — participants
need to read instructions quickly and contribute without friction.

| Level               | Size    | Weight    | Use Case                                        |
| ------------------- | ------- | --------- | ----------------------------------------------- |
| Workshop Title      | 36-42pt | Bold      | Workshop name in the title banner               |
| Workshop Subtitle   | 18-20pt | Regular   | Date, facilitator, participant count            |
| Phase Title         | 28-32pt | Bold      | Phase names ("1. EMPATHIZE", "SILENT IDEATION") |
| Activity Title      | 20-22pt | Bold      | Sub-activity names within phases                |
| Instructions        | 14-16pt | Regular   | Facilitator instructions for participants       |
| Timing              | 12-14pt | Semi-bold | Duration labels ("10 min", "5 min silent")      |
| Sticky Note Content | 14-16pt | Regular   | Pre-filled example sticky notes                 |
| Metadata            | 12pt    | Regular   | Small labels, counts, annotations               |
| Facilitator Notes   | 12-14pt | Regular   | Facilitator-only guide text                     |
| Ground Rules        | 14pt    | Regular   | Posted rules and norms                          |
| Action Item Text    | 14-16pt | Regular   | Action table content                            |
| Summary/Count       | 12pt    | Regular   | "15 ideas generated", "36 total votes"          |

### Typography Rules for Workshop Boards

1. **Phase titles are landmarks.** They must be readable at 30% zoom because the
   facilitator uses them to navigate the board and direct participant attention.

2. **Instructions should be calm, not shouting.** Use regular weight, secondary color,
   and moderate size (14pt). Instructions that are too large compete with participant
   content.

3. **Timing is always in accent color.** Duration labels use Electric Blue (or the
   palette accent) to stand apart from instructional text. Time is the most important
   metadata on a workshop board.

4. **Sticky note text is 14-16pt.** Larger is better — participants typing on sticky
   notes should not have to squint. The default Miro sticky note size works well.

5. **Do not use decorative fonts.** Workshop boards use the platform default font
   (Open Sans in Miro) for everything. No display fonts, no serif fonts, no
   handwriting fonts. Clarity over personality.

### Core Workshop Elements

| Element Type      | Miro Element         | Workshop Usage                            |
| ----------------- | -------------------- | ----------------------------------------- |
| Phase frame       | Frame                | Contains one phase of the workshop        |
| Participant zone  | Frame or Rectangle   | Bounded area where participants add notes |
| Sticky note       | Sticky Note          | Primary participant contribution element  |
| Instruction text  | Text block           | Facilitator instructions, activity rules  |
| Phase arrow       | Connector or Shape   | Flow direction between phases             |
| Timeline bar      | Segmented rectangle  | Shows full process and timing             |
| Vote dot          | Circle shape (small) | Dot voting indicator                      |
| Facilitator guide | Frame (locked)       | Facilitator-only reference panel          |
| Ground rules      | Sticky notes or text | Posted norms for the session              |
| Parking lot       | Frame                | Captures tangential ideas                 |
| Action table      | Shaped text or table | Structured action item capture            |
| Timer placeholder | Shape + text         | Visual timer reference                    |

### When to Use Sticky Notes vs. Text Blocks vs. Shapes

| Content Type            | Use This Element       | Why                                        |
| ----------------------- | ---------------------- | ------------------------------------------ |
| Participant ideas       | Sticky note            | Movable, color-coded, editable             |
| Facilitator instruction | Text block             | Locked, structured, not accidentally moved |
| Phase title             | Text block             | Fixed, bold, part of the structure         |
| Voting dots             | Small circles          | Stackable on sticky notes                  |
| Cluster borders         | Rounded rectangle      | Groups related sticky notes visually       |
| Flow arrows             | Connector/shape        | Shows process direction                    |
| Questions/prompts       | Sticky note (lavender) | Visually distinct from participant notes   |
| Templates (blank cards) | Shape (dashed border)  | Invites participant fill-in                |
| Action items            | Sticky note (coral)    | Movable, assignable                        |

### Specialized Workshop Elements

**Timer Indicator**:
A visual element showing the time remaining for the current activity:

```
+-- Timer --------+
| 10:00 remaining |
| [===----]       |
| Phase 3 of 5   |
+-----------------+
```

Implementation: A small shape (200x80px) with text, placed near the current phase.
Color: Electric Blue background, white text.

**Dot Voting Visualization**:
Blue circles (12-16px diameter) placed on or near sticky notes to indicate votes:

```
+-- Sticky Note --------+   +-- Sticky Note --------+
| "Progressive profiling"|   | "Gamification badge"  |
| ...                    |   | ...                   |
| (dot)(dot)(dot)(dot)   |   | (dot)(dot)            |
| 4 votes                |   | 2 votes               |
+------------------------+   +-----------------------+
```

**Impact/Effort Matrix**:
A 2x2 grid with labeled axes for prioritization voting:

```
        HIGH IMPACT
            |
  Quick     |    Big Bets
  Wins      |
            |
 -----------+----------- HIGH EFFORT
            |
  Filler    |    Money Pit
            |
            |
        LOW IMPACT
```

Implementation: Large shape (700x700px) with axis lines and quadrant labels.
Each quadrant is a droppable zone for sticky notes or cluster labels.

**Rotation Arrow (World Cafe)**:
Curved arrows connecting table frames in clockwise order:

```
Table 1 -----> Table 2
   ^              |
   |              v
Table 4 <----- Table 3
```

Implementation: Miro connectors with curved paths, accent color, 3px width.

## Contrast and Emphasis

### Creating Focus on Workshop Boards

Workshop boards need a clear visual hierarchy that prioritizes participant content
over facilitation structure:

1. **Current phase is the brightest zone.** If using progressive revelation or
   facilitator-guided navigation, the active phase should have full saturation
   while other phases are slightly desaturated.

2. **Participant content is the MOST visible.** Sticky notes should be brighter and
   more saturated than background frames. Yellow sticky notes on a soft cream
   background create strong figure-ground contrast.

3. **Instructions fade into the background.** Facilitator text should be readable
   but not dominant — use secondary colors (grey, muted accent) at smaller sizes.

4. **Action items are the visual endpoint.** The Action Plan frame should be the
   most structurally prominent zone — clear border, white background, structured
   layout. It is the "destination" of the entire workshop.

### Visual Weight Hierarchy

```
Highest weight (most visible):
  1. Participant sticky notes (bright, saturated, numerous)
  2. Phase titles (large, bold, primary color)
  3. Action items (structured, bordered, coral/green)
  4. Vote dots and counts (accent color, numerical)
  5. Timeline bar (accent color, segmented)
  6. Instructions (secondary color, regular weight)
  7. Ground rules (muted, background)
  8. Facilitator notes (smallest, locked, corner)
Lowest weight (least visible)
```

## Accessibility Requirements

### Workshop Board Accessibility

Workshop boards must be maximally accessible because workshops are participatory —
everyone in the room or virtual session needs to interact with the board.

1. **Sticky note contrast**: All sticky note colors must provide sufficient contrast
   for black text (#212121). The lightest standard sticky colors (yellow #FFF9C4,
   mint #E0F2F1) meet WCAG AA against dark text, but verify custom colors.

2. **Color is never the only signal**: Every color-coded element must also have a
   text label. Green sticky notes in a retro should be labeled "What Went Well"
   (in the zone header), not just be green with no explanation.

3. **Font sizes for remote participation**: If participants join via Zoom/screen
   share, text below 14pt becomes unreadable. Minimum 14pt for any participant-facing
   text. Facilitator-only notes can be 12pt.

4. **Zoom-friendly layout**: Ensure the board works when a facilitator shares their
   screen via video conference. Important content should be visible at 50-70% zoom.
   Avoid layouts that require constant panning.

5. **Screen reader consideration**: For teams with visually impaired members, provide
   a text-based summary document alongside the Miro board. The visual board cannot be
   fully screen-reader accessible, but a companion document can be.

6. **Motor accessibility for voting**: Miro's built-in voting feature is more
   accessible than manually placing dot shapes. Recommend the native voting tool
   for participants with motor impairments.

7. **Colorblind-safe sticky notes**: Avoid relying on red vs. green distinction
   alone. Use labels, shapes, or position (e.g., "Went Well" column vs. "Improve"
   column) to distinguish categories.

## Animation and Progressive Revelation

### Guiding Attention During Live Workshops

In a live facilitated workshop (in-person or remote), the facilitator needs to
direct participant attention to the current activity. Visual techniques for this:

**Zooming**: The facilitator zooms the Miro viewport to frame only the current
phase. This naturally hides future phases and prevents distraction.

**Highlighting**: Use Miro's built-in follow-me mode so all participants see the
facilitator's viewport.

**Progressive unlock**: Keep future phase frames collapsed or at reduced opacity
until the facilitator is ready to reveal them.

**Spotlight**: During discussion, zoom to a specific cluster or sticky note to
focus the conversation.

## Visual Consistency Checklist for Workshop Boards

1. All phase frames use the same height and consistent border styling
2. All sticky notes within the same zone use the same color (unless intentionally varied)
3. Timing labels are consistently formatted ("X min" format, accent color, same size)
4. Instructions follow the same format: purpose line, then rules line, then time line
5. Phase titles use consistent numbering and capitalization
6. The title banner spans full width and uses the primary color
7. Facilitator notes are visually distinct from participant areas
8. Vote dots are uniform size and color across the entire board
9. Action items follow a consistent format (Owner | Action | Due Date | Status)
10. Parking lot is present and consistently positioned (usually bottom-left or bottom-right)
11. Ground rules are visible but not dominant
12. Flow arrows between phases are consistent in size, color, and style
13. The board works at 30% zoom (overview), 50% zoom (activity), and 100% zoom (detail)
14. Color coding is consistent — the same color means the same thing everywhere on the board
15. Empty participant zones are clearly marked as "drop zone" areas with dotted borders or labels
