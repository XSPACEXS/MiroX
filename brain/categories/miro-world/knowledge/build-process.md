# Build Process — Miro World Product Showcase

## Advanced Build Workflow

Building a World Product Showcase board is a multi-phase process that requires careful planning before any shapes are placed. This guide outlines the step-by-step workflow from blank canvas to finished showpiece.

---

## Phase 1: Information Architecture (Before Opening Miro)

**Time: 30-60 minutes**

### Step 1: Define the Metaphor

- Choose the spatial metaphor (city, campus, landscape, archipelago)
- Determine how many zones/districts are needed (5-7 recommended)
- Name each zone and assign it a color from the palette
- Decide on the central hub concept

### Step 2: Gather Content

- Collect all product information that will be mapped: features, metrics, team info, roadmap
- Organize content by zone — each zone should have 3-5 "buildings" worth of content
- Identify the 4-6 key metrics that will appear on the board
- Determine sticky note content: current priorities, recent launches, known risks

### Step 3: Plan the Layout

- Sketch the zone arrangement on paper or in a simple drawing tool
- Determine which layout pattern to use (isometric city grid, campus corridor, hub-and-spoke, etc.)
- Identify the main pathways and what they represent
- Plan the presentation path (the order in which zones will be toured)

---

## Phase 2: Board Foundation (The Canvas)

**Time: 15-20 minutes**

### Step 4: Create the Board

- Set canvas size: 5000 x 3500px for standard boards
- Set background color: Deep Navy (#1A1A2E) for dark world, Warm White (#FAFAF8) for light world
- Enable grid alignment (Miro settings) for consistent placement

### Step 5: Place the Title Banner

- Create a full-width frame (4800 x 400px) at the top
- Set background to primary color
- Add the board title (56-64pt, bold, white)
- Add subtitle with product name and date
- Add decorative horizontal line

### Step 6: Create Zone Placeholder Frames

- Create empty frames for each zone at their planned positions
- Set each frame's background to the zone color at 10% opacity
- Label each frame with the zone title
- Verify spacing is consistent (200-300px gaps)

### Step 7: Place the Legend/Footer

- Create a full-width frame (4800 x 300px) at the bottom
- Leave it empty for now — it will be filled after all zones are complete

---

## Phase 3: Zone Construction (The Buildings)

**Time: 45-90 minutes per zone**

### Step 8: Build Each Zone (Repeat for Each)

**Zone header:**

- Zone title (32-42pt, bold, zone color)
- Zone description (14-16pt, charcoal)

**Buildings (3-5 per zone):**

- Create rectangular shapes for each feature/module
- Fill with zone color
- Add building label (bold) and description text (regular)
- Add key metric within or below each building
- Optionally add the isometric 3-face treatment to hero buildings

**Status stickies (2-4 per zone):**

- Add sticky notes color-coded to match zone
- Content: Priority items, recent launches, risks, shipped items
- Position stickies near the relevant building or at the zone edge

**Team info:**

- Small text block at the bottom of the zone with team lead, headcount, and cadence

### Step 9: Build the Central Hub

- Create the hub frame with distinct treatment (white background, golden border, or special shape)
- Add the product vision statement
- Add the north star metric
- Add the roadmap summary
- Add OKR status
- Add leadership info

---

## Phase 4: Connectivity (The Roads)

**Time: 20-30 minutes**

### Step 10: Draw Main Pathways

- Create thick (5px) golden yellow connectors between major zones
- Use bidirectional arrows for primary relationships
- Label each pathway with what flows along it
- Ensure pathways form a complete network (no isolated zones)

### Step 11: Draw Secondary Connections

- Create thin (2px) dashed connectors for cross-zone dependencies
- Use mint green or light gray for these connections
- Label each with the specific dependency relationship
- These connect specific buildings across zones, not just zone-to-zone

---

## Phase 5: Atmosphere (The World)

**Time: 20-40 minutes**

### Step 12: Add Decorative Elements

- Place 3-5 clouds in the sky area (top of board), white at 15-20% opacity
- Place trees along pathways (every 200-300px)
- Place 8-12 tiny human figures near buildings and along pathways
- Place stars (dark boards only) in the sky area
- Place any zone-specific decorations (icons, symbols)

### Step 13: Lock Decorative Elements

- Select all decorative elements and lock them (Miro: right-click > Lock)
- This prevents accidental movement during collaborative use
- Verify decorative elements do not obstruct text or functional elements

---

## Phase 6: Polish (The Details)

**Time: 15-30 minutes**

### Step 14: Complete the Legend

- Add color-coded squares for each zone
- Add status key (green = shipped, yellow = risk, pink = blocked)
- Add version and date information
- Add board owner and update cadence

### Step 15: Create Presentation Frames

- In Miro, create frames around key areas for presentation mode
- Frame order: Title > Central Hub > Zone 1 > Zone 2 > ... > Zone N > Legend
- Each frame should be self-contained (readable without seeing other frames)

### Step 16: Quality Review

- Zoom to full board view — does it look like a beautiful world?
- Zoom to zone view — can you read zone titles and building labels?
- Zoom to building view — is all detail text readable and accurate?
- Check all connectors are properly anchored
- Check all text is horizontal (not rotated)
- Check color consistency within each zone
- Verify the legend matches the actual board content

---

## Phase 7: Launch & Maintenance

### Step 17: Share and Onboard

- Share the board with stakeholders
- Walk through the board in a team meeting using presentation mode
- Assign zone owners for ongoing maintenance
- Set up the update cadence (weekly stickies, monthly metrics, quarterly structure)

### Step 18: Ongoing Maintenance

- Weekly: Zone owners update sticky notes and status indicators
- Monthly: Metrics refreshed, new items added
- Quarterly: Full board review — add/remove zones, update structure
- Annually: Consider a full board redesign if the product has changed significantly

---

## Time Estimates

| Phase                               | Standard Board  | Complex Board    |
| ----------------------------------- | --------------- | ---------------- |
| Information Architecture            | 30 min          | 60 min           |
| Board Foundation                    | 15 min          | 20 min           |
| Zone Construction (per zone)        | 45 min          | 90 min           |
| Zone Construction (5-7 zones total) | 3.75 - 5.25 hrs | 7.5 - 10.5 hrs   |
| Connectivity                        | 20 min          | 30 min           |
| Atmosphere                          | 20 min          | 40 min           |
| Polish                              | 15 min          | 30 min           |
| **Total**                           | **5 - 7 hours** | **9 - 12 hours** |

---

## Build Order Tip

Build the most important zone FIRST and get it to final quality. This sets the standard for all other zones. If you build all zones at draft quality first, you risk inconsistency and the temptation to "finish later" (which rarely happens).

## Collaboration During Build

If multiple people are building the board simultaneously:

- One person owns the board structure (frames, pathways, decorative elements)
- Each zone owner builds their own zone's content
- The structure owner reviews each zone for consistency before merging
- Lock completed zones to prevent accidental changes while others are still building
