# What's New Showcase

## Overview

- **Purpose**: A release announcement board styled as an interactive landscape where each feature release is a "building" or "monument" in a vibrant world. Inspired by Miro's own "What's New" boards, this template transforms changelog updates into a visually engaging, explorable showcase that teams and customers can navigate. Each release zone includes feature details, adoption metrics, user feedback, and links to documentation.
- **Best For**: Product marketing teams announcing releases, product managers sharing quarterly updates, developer relations teams showcasing API changes, customer success teams walking clients through new capabilities.
- **Complexity**: Advanced
- **Board Size**: 5000 x 3500px

## Board Layout

The board is styled as a nighttime skyline/landscape. A glowing title banner sits at top. Below, release zones are arranged as illuminated buildings along a golden pathway. A timeline ribbon runs along the bottom. "NEW" badges glow on the latest items.

## Variations

1. **Quarterly Release Board**: Combine 3 months into one board with a wider timeline. Use sub-frames within each zone for monthly breakdowns.
2. **Customer-Facing Release Notes**: Remove internal metrics, beta details, and deprecation technical notes. Focus on user benefits with screenshots and video embed links.
3. **Engineering Release Board**: Add zones for Infrastructure Changes, Performance Benchmarks, Security Patches, and Tech Debt Reduction. Include PR counts and deploy metrics.
4. **API Changelog**: Replace feature zones with Endpoints Added, Endpoints Modified, Breaking Changes, and SDK Updates. Include code snippet stickies.

## Build Instructions

1. Create the board at 5000 x 3500px with Midnight Blue (#0F0E2E) background for the "night sky" effect.
2. Place the title banner frame (4800 x 400px) at the top. Add glowing text and decorative star shapes.
3. Create the Major Release frame (1500 x 1100px) at upper-left. Add the "NEW" badge in Hot Pink, feature details, and adoption metrics.
4. Create Improvements frame (1500 x 1100px) at upper-center. Fill with 8 enhancement stickies.
5. Create Bug Fixes frame (1200 x 1100px) at upper-right. Add fix stickies and stability metrics.
6. Create Beta & Preview frame (1500 x 900px) at lower-left. Add beta feature cards and participation stats.
7. Create Deprecations frame (1200 x 900px) at lower-center. Add deprecation notices with timeline.
8. Create Coming Soon frame (1200 x 900px) at lower-right with neon cyan border.
9. Place the Release Timeline ribbon (4800 x 300px) at the bottom with milestone circles.
10. Draw golden pathway connectors between all frames.
11. Add neon cyan dashed lines for cross-zone relationships.
12. Scatter decorative star and cloud shapes across the dark background for the "nightscape" feel.

## Tips & Best Practices

- Update this board monthly and keep an archive of past months — it becomes a visual product history.
- Use Miro's presentation mode to walk through the board during monthly product update meetings.
- Add reaction emojis on stickies to let stakeholders signal excitement or concerns.
- Link each release item to its detailed spec board, Jira epic, or documentation page.
- Share a read-only link with customers for transparent release communication.
- Use the "Coming Soon" zone to build anticipation and gather early feedback from internal stakeholders.
- Pin the "NEW" badge to items shipped in the last 7 days — remove it after 2 weeks to keep the board fresh.
