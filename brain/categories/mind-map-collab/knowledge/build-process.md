# Build Process — Mind Map Product Collaboration

## Advanced Build Workflow

Mind Map Product Collaboration boards have a unique build dynamic: they are partially pre-built (the structure) and partially built live (the content during sessions). This guide covers both the pre-session preparation and the live facilitation workflow.

---

## Phase 1: Pre-Session Preparation

**Time: 20-45 minutes**

### Step 1: Define the Purpose

- Determine which template to use: Brainstorm, Product Mind Map, Research Cluster, or Decision Tree
- Write the central question or topic (be specific and measurable)
- Define the expected outcome (ideas, decisions, actions, research synthesis)
- Determine participant count and session duration

### Step 2: Prepare the Board Structure

**For Brainstorm Boards:**

- Create the board (5000 x 3500px, Soft Lavender background)
- Place the title banner with the question
- Create personal zones (one per participant, color-coded)
- Create the empty synthesis center (large white frame in the center)
- Create the voting panel, priority matrix, and action items frames (empty)
- Add phase timer labels

**For Product Mind Maps:**

- Create the board (5500 x 4000px, Soft Gray background)
- Place the central product node with product name and key metrics
- Create empty branch frames at planned positions (6 branches for standard)
- Add branch labels and colors
- Create the mockup showcase strip (empty)

**For Research Cluster Maps:**

- Create the board (5000 x 4000px, Warm White background)
- Place the research header with study information
- Create the raw data wall (empty, with section labels for data types)
- Create 5 cluster frames (empty, with color assignments)
- Create insights summary and recommendations frames (empty)

**For Decision Trees:**

- Create the board (5000 x 3500px, Soft White background)
- Place the decision context banner
- Create the root decision diamond
- Create placeholder frames for each main branch
- Create criteria matrix, risk register, and recommendation panels (empty)

### Step 3: Pre-Populate Known Content

- For product mind maps: Add known features, personas, integrations before the session
- For research maps: Upload raw data (quotes, stats) to the data wall before synthesis
- For decision trees: Document the options and known criteria before the evaluation session
- For brainstorms: Do NOT pre-populate ideas — that defeats the purpose

### Step 4: Test the Board

- Open the board on both desktop and mobile
- Verify all frames are positioned correctly
- Verify the question/topic is clear and readable
- Ensure there is enough space for growth (boards should not feel pre-filled)

## Phase 2: Session Facilitation (Live Build)

### For Brainstorm Sessions

**Phase 2A: Solo Ideation (10 minutes)**

- Display the timer and start the countdown
- Participants write ideas in their personal zones
- Facilitator does NOT participate — monitors energy and time
- Music playing (optional) to reduce self-censoring
- NO discussion during this phase

**Phase 2B: Share & Cluster (15 minutes)**

- Each participant shares their top 3-5 ideas verbally
- As they share, they (or the facilitator) drag stickies to the synthesis center
- Group similar ideas together as they arrive
- Name the clusters as they form (theme labels)
- The facilitator asks: "Does this idea belong with this cluster, or is it a new theme?"
- Unclassifiable ideas go in an "Outliers" area for later discussion

**Phase 2C: Vote & Prioritize (10 minutes)**

- Explain the voting rules: 5 dots per person, multiple votes on one idea allowed
- Start the voting timer
- Participants place dot votes on their preferred ideas or clusters
- After voting, the facilitator tallies results and highlights the top 3-5 ideas
- Move top ideas into the priority matrix (Impact vs Effort 2x2)

**Phase 2D: Action Planning (10 minutes)**

- For each top-voted idea, define:
  - Owner: Who will drive this forward?
  - First step: What is the immediate next action?
  - Due date: When will the first step be complete?
  - Success metric: How will we know it worked?
- Document action items in the Action Items frame
- Assign and confirm with each owner

### For Product Mind Map Sessions

**Build sequence (60-90 minutes):**

1. Start at the central node — validate the product vision and key metrics
2. Build one branch at a time, clockwise from the top
3. For each branch: present the known content, then open for discussion and additions
4. Draw cross-connections as they are discovered during discussion
5. Assign each branch to an owner for ongoing maintenance
6. Review the complete map at the end — identify gaps and next steps

### For Research Synthesis Sessions

**Build sequence (90-120 minutes):**

1. Start with the raw data wall — review all evidence
2. Begin affinity mapping: drag evidence to cluster frames
3. For each cluster: discuss what the evidence means together
4. Write insight statements as a group (facilitator drafts, group refines)
5. Identify cross-cluster connections
6. Rank insights by impact and confidence
7. Develop recommendations from the top insights
8. Assign recommendation owners and timelines

### For Decision Tree Sessions

**Build sequence (60-90 minutes):**

1. Start with the decision context — ensure all participants agree on the question
2. Present the main options (branches) — typically pre-populated
3. For each branch: discuss sub-decisions and criteria
4. Evaluate each outcome against the weighted criteria
5. Populate the criteria matrix as a group
6. Identify and document risks
7. Converge on a recommendation
8. Document next steps and decision record

---

## Phase 3: Post-Session Processing

**Time: 15-30 minutes**

### Step 5: Clean Up the Board

- Align stickies within clusters (neat up the mess from live clustering)
- Ensure all text is readable and properly formatted
- Fix any broken connectors or misplaced elements
- Add any missed cross-connections discussed but not drawn during the session

### Step 6: Document the Outcomes

- Verify all action items have owners and dates
- Add a "Session Summary" text block capturing key decisions
- Add a timestamp: "Session completed: [Date], [Participants]"

### Step 7: Share and Follow Up

- Share the board link with all participants
- Share the Action Items frame as a standalone view with stakeholders
- Set calendar reminders for action item due dates
- Schedule the next session if part of a series

### Step 8: Archive or Evolve

- **Brainstorm boards**: Archive after action items are captured. Duplicate if a follow-up session is planned.
- **Product mind maps**: Keep as a living document. Set monthly update reminders.
- **Research cluster maps**: Archive after recommendations are delivered. Keep for reference during implementation.
- **Decision trees**: Lock and archive after the decision is made. Add a "DECIDED" watermark with the date and outcome.

## Facilitation Best Practices

1. **Start on time, end on time**: Strict time-boxing prevents sessions from dragging
2. **Silent before social**: Individual ideation before group discussion reduces groupthink
3. **Facilitate, do not contribute**: The facilitator manages the process, not the content
4. **Make it physical**: Encourage participants to physically drag their stickies (even digitally, the act of moving matters)
5. **Name the elephant**: If the group is stuck or avoiding a topic, call it out
6. **Celebrate the mess**: The divergent phase SHOULD look chaotic. Reassure participants that order will come
7. **Capture everything**: Even rejected ideas have value — dim them, do not delete them
8. **End with commitment**: Every session must end with named owners and specific next actions
