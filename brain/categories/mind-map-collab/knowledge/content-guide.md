# Content Guide — Mind Map Product Collaboration

## Domain Knowledge: Collaborative Thinking Content

### What Belongs on a Mind Map Collaboration Board

Mind map boards capture the full lifecycle of collaborative thinking: from raw ideas through structured synthesis to committed actions. The content ranges from single-word brainstorm concepts to fully articulated research findings with evidence and recommendations.

**The five content layers:**

1. **The Question**: The central problem, opportunity, or topic being explored
2. **Raw Ideas**: Unfiltered contributions from individual participants
3. **Structured Themes**: Clustered, labeled groupings of related ideas
4. **Evaluated Insights**: Synthesized findings with evidence and confidence ratings
5. **Committed Actions**: Prioritized decisions with owners, dates, and success criteria

## Content Templates by Board Type

### Brainstorm Board Content

#### Central Question

```
Primary Question: "How might we [verb] [subject] by [measure] in [timeframe]?"
Context: [1-2 sentences explaining why this question matters now]
Session Info: [Sprint/Team Name] | [Date] | [Duration] | [Participant Count]
```

**Good question formats:**

- "How might we reduce customer churn by 30% in the next two quarters?"
- "What are all the ways we could improve our onboarding experience?"
- "What would our product look like if we had unlimited engineering resources?"
- "How should we respond to [Competitor X]'s new feature launch?"

**Bad question formats (too vague):**

- "How do we grow?" (Grow what? By how much? By when?)
- "What should we build?" (For whom? To solve what problem?)
- "Ideas?" (No framing at all)

#### Personal Zone Content

Each participant's zone contains 5-8 sticky notes with individual ideas:

```
Sticky Format: [1-2 sentence idea]
Guidelines:
- One idea per sticky note
- Be specific enough that someone else can understand the idea
- Include a "how" if possible, not just a "what"
- No filtering — every idea is valid in divergent phase
```

**Good sticky examples:**

- "Proactive outreach when usage drops below 3 sessions/week — automated email from CSM"
- "In-app health score visible to customers — transparency builds trust and self-service"
- "Gamify feature adoption with achievement badges — reward exploration"

**Bad sticky examples:**

- "Better onboarding" (too vague — better how?)
- "Fix bugs" (not an ideation-level concept)
- "We already tried this" (judgment, not ideation)

#### Synthesis Center Content

The shared mind map in the center uses clustered themes:

```
Theme Cluster Name: [2-4 word descriptive label]
Ideas in Cluster: [Count]
Sub-ideas:
- [Specific idea from participant zone]
- [Specific idea from participant zone]
- [Related idea]
- [Related idea]
```

### Product Mind Map Content

#### Central Product Node

```
Product Name: [Name]
Tagline: "[One-line value proposition]"
Key Metrics: [WAU/DAU] | [Teams/Customers] | [ARR/MRR]
Version: [Current version] | [Date]
```

#### Feature Branch Content

```
Feature Category: [Category Name]
Feature Count: [Number] total capabilities

Feature Node:
- Name: [Feature Name]
- Description: [What it does in 1-2 sentences]
- Status: [GA / Beta / Alpha / Planned]
- Key Metric: [Usage stat or performance metric]
- Related Persona: [Which persona uses this most]
```

#### Persona Branch Content

```
Persona Card:
- Name: "[Persona Name]" (e.g., "Product Paula")
- Role: [Job title]
- Company Profile: [Company type and size]
- Goals: [2-3 bullet points]
- Pain Points: [2-3 bullet points]
- Key Features Used: [List of product features this persona relies on]
- Conversion Note: [Where this persona comes from, what convinces them]
```

#### Integration Branch Content

```
Integration Category: [Category Name]

Integration Node:
- Name: [Integration Name]
- Type: [Native / API / Marketplace / Zapier]
- Status: [Live / Beta / Planned]
- Connected Instances: [Number if live]
- Key Use Case: [What users do with this integration]
```

### Research Cluster Map Content

#### Raw Data Content Types

**Interview Quote:**

```
"[Verbatim quote from participant]"
— Participant [ID], [Role/Title], [Company Type]
Interview Date: [Date] | Duration: [Minutes]
```

**Survey Data Point:**

```
[Metric Name]: [Value]
Source: [Survey Name], [Date]
Sample Size: n=[Number]
Confidence: [Confidence interval if applicable]
```

**Observation:**

```
Observed: [What was seen or recorded]
Context: [During what activity, at what point]
Source: [Contextual inquiry / Usability test / Field study]
Date: [Date] | Observer: [Name]
```

#### Evidence Card Content

```
Evidence Type: [Interview / Survey / Observation / Analytics]
Content: [Quote, statistic, or observation]
Source: [Participant ID, survey name, or data source]
Relevance: [Why this evidence matters to the cluster]
Cluster Assignment: [Theme name]
```

#### Insight Statement Content

```
Insight: [Complete sentence stating the finding]
Confidence: [High / Medium / Low]
Evidence Count: [Number] pieces of evidence supporting this insight
Evidence Types: [Interview: X, Survey: Y, Observation: Z]
Implication: [What this means for the product/team]
```

#### Recommendation Content

```
Recommendation: [Action statement]
Supported By: Insight [#] — [Brief insight reference]
Confidence: [High / Medium / Low]
Effort: [Low / Medium / High]
Timeline: [Quarter or specific date range]
Owner: [Name or team]
Success Metric: [How we will know this recommendation worked]
```

### Decision Tree Content

#### Decision Context

```
Decision Statement: "[Should we X or Y?]"
Context: [Why this decision needs to be made now]
Decision Makers: [Names and roles]
Deadline: [Date]
Impact: [What is at stake — financial, strategic, operational]
Reversibility: [How easy is it to undo this decision]
```

#### Decision Node Content

```
Decision: [Question being decided]
Options: [List of possible paths]
Criteria for Decision:
- [Criterion 1]: [How we evaluate it]
- [Criterion 2]: [How we evaluate it]
- [Criterion 3]: [How we evaluate it]
```

#### Outcome Node Content

```
Outcome: [Name of this outcome]
Assessment: [Positive / Neutral / Negative]
Cost: $[Estimated cost]
Timeline: [Estimated time to realize]
Risk Level: [Low / Medium / High]
Customization: [Low / Medium / High]
Maintenance: [FTE or hours per month]
Probability of Success: [%]
Reversibility: [Easy / Moderate / Difficult / Irreversible]
```

## Writing Style for Mind Maps

### Tone

- **Exploratory during divergent phases**: "What if we..." "Imagine..." "Consider..."
- **Analytical during convergent phases**: "The data suggests..." "Based on X evidence..."
- **Decisive during action phases**: "We will..." "Owner: [Name] by [Date]"
- **Non-judgmental in personal zones**: No idea is criticized during ideation

### Node Text Guidelines

| Node Type                | Text Length | Writing Style                               |
| ------------------------ | ----------- | ------------------------------------------- |
| Central question         | 10-20 words | Question format, specific and measurable    |
| Sticky note (brainstorm) | 5-20 words  | Active voice, specific, one idea per sticky |
| Theme cluster label      | 2-4 words   | Noun phrase describing the theme            |
| Insight statement        | 15-30 words | Complete declarative sentence               |
| Evidence card            | 15-40 words | Quote (verbatim) or stat (with source)      |
| Recommendation           | 15-25 words | Imperative sentence (action-oriented)       |
| Action item              | 10-20 words | Task + owner + date                         |
| Decision criteria        | 5-15 words  | Conditional statement ("If X, then Y")      |

## Facilitation Content

### Phase Transition Prompts

Content for the facilitator to display during phase changes:

**Phase 1 (Solo Ideation) prompt:**
"Take 10 minutes to silently generate ideas in your personal zone. One idea per sticky note. Quantity over quality. No discussion yet."

**Phase 2 (Share & Cluster) prompt:**
"Share your top ideas with the group. As you share, drag your stickies to the Synthesis Center. Group similar ideas together. Name the clusters."

**Phase 3 (Vote & Prioritize) prompt:**
"You have 5 votes. Place your dots on the ideas you think have the highest potential. You can put multiple votes on one idea."

**Phase 4 (Action Planning) prompt:**
"For the top-voted ideas, define: Who owns it? What is the first step? When will it be done? How will we know it worked?"

## Content Refresh Cadence

| Board Type           | Update Frequency                        | When to Archive                       |
| -------------------- | --------------------------------------- | ------------------------------------- |
| Brainstorm board     | Updated during session, then archived   | After action items are captured       |
| Product mind map     | Monthly updates                         | Never (living document)               |
| Research cluster map | Updated during synthesis, then archived | After recommendations are delivered   |
| Decision tree        | Updated until decision is made          | After decision is made and documented |
