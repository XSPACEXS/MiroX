# Sales Board Visual Guide

## Introduction

Sales boards live in a unique visual territory. They must feel professional enough for board-of-directors presentations yet dynamic enough for weekly pipeline scrums. They must communicate urgency without creating anxiety. They must present dense data without overwhelming. This guide covers the complete visual language for god-level sales Miro boards: color palettes, typography, iconography, element choices, and the reasoning behind every decision.

## Color Palettes

### Palette 1: Navy Authority (Default Sales Palette)

The flagship palette for sales boards. Navy conveys trust, authority, and seriousness — exactly the tone a revenue organization needs. The green accent connects to money and positive outcomes. Steel blue provides a professional mid-tone for secondary elements.

| Role           | Color Name    | Hex     | Usage                                           |
| -------------- | ------------- | ------- | ----------------------------------------------- |
| Primary        | Navy Blue     | #1B2A4A | Headers, borders, primary text, stage labels    |
| Secondary      | Steel Blue    | #4A6FA5 | Secondary text, connectors, card borders        |
| Accent         | Emerald Green | #2E7D32 | Won deals, positive metrics, growth indicators  |
| Warning        | Amber Gold    | #F9A825 | At-risk indicators, cautionary notes, attention |
| Danger         | Crimson       | #C62828 | Lost deals, negative metrics, missed targets    |
| Background     | Light Slate   | #F0F2F5 | Board background, empty areas                   |
| Surface        | White         | #FFFFFF | Card backgrounds, panel fills                   |
| Text Primary   | Charcoal      | #1A1A2E | Headlines, card titles                          |
| Text Secondary | Gray          | #6B7280 | Labels, descriptions, metadata                  |

**When to use**: Default for pipeline boards, deal trackers, forecast dashboards, and any revenue-focused board.

**Reasoning**: Navy is the most common color in corporate logos and executive presentations because it signals competence and reliability. Emerald green taps into the universal association of green with growth, money, and success. The crimson red creates sufficient contrast against the blue spectrum to signal danger without clashing.

## Typography Rules

### Font Size Hierarchy for Sales Boards

| Level          | Size    | Weight            | Color                               | Usage                             |
| -------------- | ------- | ----------------- | ----------------------------------- | --------------------------------- |
| Board Title    | 36px    | Bold              | White on primary                    | Main board name in header strip   |
| Section Header | 22-24px | Bold              | Primary color                       | Frame titles                      |
| Sub-Header     | 18px    | Bold              | Primary or text primary             | Sub-sections within frames        |
| KPI Value      | 28-36px | Bold              | Varies by state (green/red/neutral) | Large metric numbers              |
| Card Title     | 14-16px | Bold              | Text primary                        | Deal name, company name, rep name |
| Card Body      | 12-13px | Regular           | Text primary                        | Deal details, descriptions        |
| Card Metadata  | 11px    | Regular           | Text secondary                      | Dates, sources, secondary info    |
| Label          | 10-11px | Regular or Medium | Text secondary                      | KPI labels, column sub-headers    |
| Badge/Tag      | 10px    | Medium            | White on colored background         | Status indicators, stage tags     |

### Typography Best Practices for Sales

1. **Numbers are heroes**: KPI values and dollar amounts should be the largest text on the board after the title. Sales people look at numbers first, context second.

2. **Currency formatting**: Always include the dollar sign. Use commas for thousands. Use one decimal place for millions ($2.4M, not $2,400,000 or $2.4 million). Abbreviate: K for thousands, M for millions.

3. **Percentage formatting**: Include the percent sign. One decimal place maximum. Include direction when showing change: "+8.3%" not "8.3%".

4. **Date formatting**: Use "Mar 6, 2026" format for readability. Never use "03/06/2026" (ambiguous internationally) or "2026-03-06" (too technical for sales audience).

5. **Name formatting**: First name last name for people. Company name, not stock ticker, for accounts.

## Element Selection Guide

### When to Use Each Miro Element

| Element                | Use For                                                     | Do Not Use For                                          |
| ---------------------- | ----------------------------------------------------------- | ------------------------------------------------------- |
| **Sticky Note**        | Deal cards, opportunity cards, objection-response pairs     | Headers, KPI displays, chart data                       |
| **Shape (Rectangle)**  | KPI cards, status banners, table cells, chart areas         | Deal cards (use stickies), text-only content            |
| **Shape (Circle)**     | Status indicators, RAG dots, pie chart segments             | Deal cards, buttons, headers                            |
| **Shape (Line/Arrow)** | Stage progression, relationships, flow direction            | Decoration, borders (use rectangles)                    |
| **Text Block**         | Headers, labels, body text, annotations                     | Data that needs visual encoding (use shapes)            |
| **Frame**              | Section containers, zoom targets, organizational boundaries | Individual card containers (too granular)               |
| **Connector**          | Explicit relationships between frames or elements           | Decoration, implied relationships                       |
| **Image**              | Company logos, profile photos, product screenshots          | Icons (use shapes or emoji), charts (build from shapes) |

### Deal Card Anatomy

Every deal card on a sales board should follow this standard anatomy:

```
+---------------------------------------+
| COMPANY NAME — Deal Title       [14px bold]
| $XXX,XXX | XX% probability      [12px, value=bold]
| Contact: First Last, Title      [12px]
| Source: Channel — Detail        [11px, gray]
| Next: Action item by DATE       [12px, bold, accent color]
|
| [Status badge]  [Age: XX days]  [10px badges]
+---------------------------------------+
```

Card color encodes the stage:

- Lead stage: Light blue (#BBDEFB)
- Qualified: Light purple (#D1C4E9)
- Proposal: Light orange (#FFE0B2)
- Negotiation: Light yellow (#FFF9C4)
- Won: Light green (#C8E6C9)
- Lost: Light red (#FFCDD2)

### KPI Card Anatomy

```

|

|

+---------------------------+
```

## Contrast and Readability Rules

### Background-to-Text Contrast Matrix

| Background           | Minimum Text Color | Notes                                                      |
| -------------------- | ------------------ | ---------------------------------------------------------- |
| White (#FFFFFF)      | #424242 or darker  | Never use light gray text on white                         |
| Light gray (#F0F2F5) | #424242 or darker  | Slightly more forgiving than white                         |
| Navy (#1B2A4A)       | #FFFFFF or #B0BEC5 | White preferred; light gray acceptable for secondary       |
| Deep teal (#006064)  | #FFFFFF or #B2DFDB | White for headlines; light teal for metadata               |
| Emerald (#2E7D32)    | #FFFFFF only       | Green on any background other than white is hard to read   |
| Crimson (#C62828)    | #FFFFFF only       | Same as emerald — always white text on colored backgrounds |

### Accessibility Minimum Standards

- All text must meet WCAG AA contrast ratio (4.5:1 for normal text, 3:1 for large text)
- RAG status indicators must include text labels, not just color dots, for colorblind users
- Deal cards must not rely solely on sticky note color to encode stage — include a text stage label
- KPI change direction must include arrows (up/down), not just green/red color

## Decorative and Structural Elements

### Dividers and Separators

Use horizontal lines sparingly to separate zones. Specifications:

- Line weight: 1-2px
- Color: Text secondary (#6B7280) or lighter (#E0E0E0)
- Style: Solid for zone separators, dashed for sub-section separators
- Never use decorative borders on every card — it creates visual noise

### Shadow and Depth

KPI cards should have subtle shadows to elevate them from the background:

- Shadow: 0px 2px 8px rgba(0,0,0,0.08)
- This creates a slight "floating card" effect that signals interactivity and importance
- Do not apply shadows to sticky notes (they already have a built-in tilt effect)
- Do not apply shadows to frames (they are structural, not interactive)

### Icons and Emoji Usage

Sales boards benefit from strategic emoji use for quick scanning:

| Element             | Recommended Icon     | Purpose                             |
| ------------------- | -------------------- | ----------------------------------- |
| Champion            | Star symbol or crown | Identifies the internal advocate    |
| Economic Buyer      | Dollar sign          | Identifies budget holder            |
| Technical Evaluator | Gear or wrench       | Identifies technical decision-maker |
| Blocker             | X mark or stop sign  | Identifies resistance               |
| Phone call          | Phone emoji          | Activity type                       |
| Email               | Envelope emoji       | Activity type                       |
| Meeting             | Calendar emoji       | Activity type                       |
| Alert               | Warning triangle     | Attention required                  |
| Won                 | Checkmark            | Completed/successful                |
| Lost                | X mark               | Failed/lost                         |
| New deal            | Sparkle or star      | Freshly added                       |
| Aging deal          | Clock or hourglass   | Stale indicator                     |

Use emoji sparingly — maximum 1-2 per card. Their purpose is rapid visual scanning, not decoration.

### Progress Bars

For quota attainment, pipeline coverage, and conversion rates, use in-line progress bars:

```
Closed Revenue:  ████████████░░░░░░░░  62%  ($744K / $1.2M)
Pipeline:        ████████████████████  3.2x coverage  ✓
Win Rate:        ██████░░░░░░░░░░░░░░  28%  (target: 32%)
```

Specifications:

- Bar height: 12-16px
- Filled color: State-appropriate (green for on-track, amber for near-miss, red for off-track)
- Empty color: #E0E0E0 (light gray)
- Value and label to the right of the bar
- Width: 150-250px depending on context

## Chart Representation on Miro Boards

Since Miro does not have native charts, sales boards represent data visually using shapes:

### Bar Chart Representation

- Use rectangles of varying widths (horizontal) or heights (vertical)
- Color-code bars by category
- Add value labels at the end of each bar
- Include axis labels and a title

### Line Chart Representation

- Use connected circle shapes (small, 8-12px diameter) for data points
- Draw line connectors between points
- Use a dashed line for targets or benchmarks
- Add value labels at key inflection points, not every point

### Pie/Donut Representation

- Use a large circle with colored arc segments (approximated with overlapping shapes)
- Always include a legend with percentages
- Limit to 5-6 segments maximum
- Order segments largest to smallest, clockwise from 12 o'clock

### Funnel Representation

- Use descending-width rectangles (trapezoids if available)
- Label each stage with count and percentage
- Show drop-off percentages between stages
- Color-code by stage health, not just position

## Visual Anti-Patterns for Sales Boards

### Anti-Pattern 1: The Rainbow Board

Using a different bright color for every element. When everything is colorful, nothing communicates meaning through color.
**Fix**: Restrict your palette to 5-7 colors maximum. Use color semantically, not decoratively.

### Anti-Pattern 2: The Wall of Text

Turning deal cards into mini-documents with paragraphs of text.
**Fix**: Card body text should be 3-5 lines maximum. Use abbreviations, icons, and short phrases. Link to CRM for full detail.

### Anti-Pattern 3: The Pixel-Perfect Illusion

Spending hours aligning elements to create a "perfect" static layout that breaks the moment someone adds a deal.
**Fix**: Design for dynamic content. Leave room for new cards. Use frames that can expand. Accept that a living board will never look as clean as a design mockup — and that is a good thing.

### Anti-Pattern 4: The Invisible Hierarchy

Giving equal visual weight to headlines, KPIs, card titles, labels, and metadata. Everything looks the same size and weight.
**Fix**: Create clear visual hierarchy with at least 4 distinct size/weight levels. The eye should know where to look first, second, and third.

### Anti-Pattern 5: The Dark Mode Sales Board

Using a dark background for the entire board. While dramatic, dark backgrounds increase cognitive load for dense data boards and reduce readability for extended viewing.
**Fix**: Use dark backgrounds only for header strips and feature sections. Keep the working area (where cards live) on light backgrounds.

## Board Finishing Touches

### The Professional Polish Checklist

Before sharing any sales board with stakeholders, verify:

1. **Alignment**: All frames are on a grid. No elements are misaligned by a few pixels.
2. **Consistency**: Every deal card follows the same format. Every KPI card uses the same template.
3. **Completeness**: No "TODO" placeholders. No empty frames. Every section has content or is explicitly labeled "Coming Soon."
4. **Readability**: Zoom out to full board view. Can you read the KPI values? Can you read the section headers? If not, increase font sizes.
5. **Color coherence**: Count the distinct colors on the board. If more than 8, simplify.
6. **White space**: Ensure at least 15-20px padding inside every frame and between every element. Cramped boards feel chaotic.
7. **Connection clarity**: Every connector has a purpose. No orphaned arrows. No unexplained lines.
8. **Timestamp**: The "last updated" date is current. The owner name is correct.
9. **Presentation mode**: Walk through the board in Miro's presentation mode. Does the frame order match the narrative?
10. **Mobile readability**: Check the board on a tablet. KPI values and headlines should still be readable.
