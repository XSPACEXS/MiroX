# Team Building Activity Board

## Overview

- **Purpose**: A collaborative board for team building sessions — mapping team values, identifying individual strengths, establishing working agreements, sharing fun facts, and aligning on collaboration preferences. Designed to be used during team offsites, kickoffs, or whenever a team needs to strengthen its dynamics.
- **Best For**: Team leads, managers, HR facilitators, and scrum masters running team offsites, new team kickoffs, quarterly retrospectives, or post-reorg bonding sessions.
- **Complexity**: Simple
- **Board Size**: 4000 x 3000px

## Board Layout

The board is arranged in a collaborative workshop format — a central team identity area surrounded by activity frames that participants contribute to simultaneously.

```
+------------------------------------------------------------------+
|              HEADER: Team Building — [Team Name]                  |
+------------------------------------------------------------------+
|                     |                      |                      |
|  Frame 1:           |  Frame 2:            |  Frame 3:            |
|  Team Values        |  Strengths           |  Working             |
|  (What we           |  Map                 |  Agreements          |
|   stand for)        |  (What we bring)     |  (How we work)       |
|                     |                      |                      |
+---------------------+----------------------+----------------------+
|                     |                      |                      |
```

## Variations

1. **Remote Team Building**: Replace the in-person workshop format with async activities spread over a week. Add a "Virtual Water Cooler" frame with Donut pairing schedules and a "Show Your Workspace" photo wall.
2. **Cross-Functional Team Kickoff**: Add a "Role Mapping" frame where each function explains what they do and what they need from others. Add a RACI matrix for key workflows.
3. **Team Retrospective**: Replace the Values and Fun Facts frames with "What Went Well," "What Didn't," and "What We'll Change" sections. Keep the Working Agreements frame for updating existing agreements.

## Build Instructions

1. Create the board at 4000 x 3000px with #FAFAFA background.
2. Build the header at position (0, 0), full width, 160px tall, #1565C0 background.
3. Create Frame 1 (Values) at position (50, 180) with size 1250 x 1200px, #F3E5F5 background. Leave space for participants to add sticky notes during the session.
4. Create Frame 2 (Strengths Map) at position (1350, 180) with size 1250 x 1200px, #E8F5E9 background. Pre-create 8 empty strength cards with just names.
5. Create Frame 3 (Working Agreements) at position (2650, 180) with size 1250 x 1200px, #FFF3E0 background. Start with blank sticky notes for participants to fill in.
6. Create Frame 4 (Fun Facts) at position (50, 1430) with size 1250 x 1200px, #FCE4EC background. Pre-populate the prompt; participants fill in during ice breaker.
7. Create Frame 5 (Collaboration Preferences) at position (1350, 1430) with size 1250 x 1200px, #E0F2F1 background. Create the table structure with names pre-filled.
8. Create Frame 6 (Action Items) at position (2650, 1430) with size 1250 x 1200px, #F5F5F5 background. Start empty; fill in during the wrap-up discussion.
9. Add connectors between frames as specified.
10. Before the session: pre-fill names and prompts. During the session: participants fill in their sticky notes. After: Mia adds the voted values and action items.

## Tips & Best Practices

- **Run this quarterly, not just once**: Team dynamics change. New members join, projects shift priorities, and agreements need refreshing.
- **Facilitate, don't dictate**: The manager should guide the process but not dominate the contributions. Every team member's voice should appear on the board.
- **Start with fun, end with commitments**: Fun Facts first warms people up. Working Agreements last ensures people are engaged enough to commit.
- **Make it physical when possible**: If you are in person, use physical sticky notes first, then digitize to the Miro board. The tactile experience builds more energy than typing.
- **Follow through on actions**: The biggest risk is that the action items die after the workshop. Schedule follow-ups and check items off publicly.
- **Keep the board visible**: Link it in the team Slack channel. Reference it when someone violates a working agreement ("Hey, per our agreement #6...").
