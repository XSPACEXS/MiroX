# Project Management — Visual Guide

## Overview

Project management boards prioritize function over form, but the best boards achieve both. Visual design in PM boards serves a singular purpose: making project information instantly scannable, unambiguously clear, and motivationally compelling. Every color, font choice, and visual element must earn its place by communicating faster than text.

**Reasoning**: A cool, authoritative palette that conveys control and reliability. The navy-and-gray foundation feels corporate without being cold, and the traffic-light status colors (green/amber/red) are instantly recognizable. ## Typography Scale

### PM-Specific Typography

Project management boards have unique typography needs: they must display a lot of information in small spaces while maintaining scanability.

| Element             | Font Size | Weight          | Usage                                 |
| ------------------- | --------- | --------------- | ------------------------------------- |
| Board Title         | 28-36px   | Bold (700)      | Top of board, project name            |
| Section Header      | 20-26px   | Semi-Bold (600) | Column headers, sprint names          |
| Metric Value        | 36-48px   | Bold (700)      | Dashboard numbers (BIG and prominent) |
| Card Title          | 14-16px   | Semi-Bold (600) | Task name on cards                    |
| Card Description    | 12-14px   | Regular (400)   | Task details, acceptance criteria     |
| Tag/Label           | 10-12px   | Medium (500)    | Category tags, team labels            |
| Date/Timestamp      | 11-13px   | Regular (400)   | Due dates, last updated               |
| WIP Counter         | 16-18px   | Bold (700)      | Column limits (3/5)                   |
| Progress Percentage | 14-16px   | Bold (700)      | Completion indicators                 |
| Annotation/Note     | 10-12px   | Regular (400)   | Facilitator notes, instructions       |

### Font Recommendations

**Primary**: Inter, Roboto, or Source Sans Pro — clean, highly legible, excellent number rendering
**Monospace** (for metrics): JetBrains Mono, Roboto Mono — aligned numbers read better in monospace
**Avoid**: Serif fonts (reduce scanability in small cards), decorative fonts, thin/ultralight weights

## Element Selection Guide

### Cards: The Core PM Element

Cards are the atomic unit. Their visual design determines the board's effectiveness.

**Standard Card (Kanban/Sprint)**:

- Width: 250-350px
- Height: 100-160px
- Background: White (#FFFFFF)
- Border: 2px, left-accented (4px left border in priority color)
- Corner radius: 8-12px
- Shadow: Very subtle (0 1px 3px rgba(0,0,0,0.12))
- Internal padding: 12-16px

**Card information hierarchy** (top to bottom):

1. Priority badge + Title (bold, 14-16px) — the reader sees this first
2. Assignee avatar + Due date — who and when
3. Tags/labels — categorization
4. Story points or effort estimate — how big

**Card visual states**:

- Default: White background, gray border
- In Progress: Blue left border (4px)
- Blocked: Red left border (4px), subtle red tint background (#FFF5F5)
- Overdue: Red border all around (2px), pulsing red dot (if animated)
- Done: Green left border, text slightly muted, optional strikethrough on title
- Selected/Focused: Blue border all around, subtle blue tint background

### Column Headers

**Design**:

- Full column width
- Height: 70-100px
- Background: Status color at 10-20% opacity
- Top border: 4px solid in status color at full saturation
- Content: Column name (bold, left-aligned) + WIP indicator (regular, right-aligned)

### Timeline Bars

**Design**:

- Height: 30-50px for individual tasks, 20-30px for sub-tasks
- Corner radius: 4-8px (consistent)
- Fill: Team or status color
- Progress: Darker shade of fill color from left edge (percent complete)
- Text: Task name inside the bar (white text on colored fill) or above (dark text)
- Hover state: Slight scale increase (102%) with tooltip showing details

### Milestone Markers

**Design**:

- Shape: Diamond (rotated 45-degree square)
- Size: 20-30px
- Color: Gold (#F9A825) for normal milestones, Red (#F44336) for critical
- Label: Date and name below the diamond, centered

## Creating Effective Status Visualization

### Traffic Light System

The three-color traffic light (green/amber/red) is the most universally understood status system. Apply it consistently:

| Status Level       | Color         | Visual Treatment                               |
| ------------------ | ------------- | ---------------------------------------------- |
| Healthy / On Track | Green #4CAF50 | Subtle background tint, small green dot        |
| Needs Attention    | Amber #FF9800 | Medium border, amber badge                     |
| Critical / Blocked | Red #F44336   | Bold border, red background tint, warning icon |

### Progress Bars

Progress bars communicate completion at a glance:

- **Height**: 8-16px for inline bars, 16-24px for standalone bars
- **Background**: Light gray (#E0E0E0) track
- **Fill**: Green gradient from left, or status-colored fill
- **Corner radius**: Half of height (fully rounded)
- **Label**: Percentage text to the right of the bar or centered inside
- **Thresholds**: 0-33% = red fill, 34-66% = amber fill, 67-100% = green fill

### Burndown/Burnup Indicators

For sprint and project views:

- Simple sparkline (150-200px wide, 40-60px tall)
- Ideal line: Dashed gray diagonal
- Actual line: Solid colored line (green if below ideal, red if above)
- No axis labels needed at this size — the trend is what matters

## Decorative Elements That Add Value

### Capacity Gauges

Semi-circular gauge (150-200px diameter) showing team or sprint capacity utilization. Green zone (0-80%), yellow zone (80-100%), red zone (100%+). Place in sprint headers or dashboard widgets.

### Completion Rings

Circular progress indicator (80-120px diameter) with the percentage in the center. Use for sprint completion, milestone progress, or overall project health.

### Divider Lines

Horizontal or vertical lines (1-2px, light gray) separating sections. Add small labels at the midpoint: "This Sprint | Next Sprint" or "Engineering | Design | QA".

### Status Badges

Small rectangular badges (60-100px wide, 24-32px tall) with rounded corners:

- "ON TRACK" (green background, white text)
- "AT RISK" (amber background, dark text)
- "BLOCKED" (red background, white text)
- "DONE" (green border, green text, transparent background)

### Today Marker

A vertical red dashed line on timeline views, with "TODAY" label and the current date. This is the most important reference point on any timeline board.

## Accessibility Guidelines

### PM-Specific Accessibility

Project boards are often displayed on large screens in common areas, projected in meetings, or viewed on mobile devices. Accessibility must account for these diverse viewing contexts.

**Large screen / war room display**:

- Minimum font size 14px (readable from 3 meters)
- High contrast status colors
- No information encoded only in color (always pair with text or icon)
- Clear card borders and column dividers

**Mobile / tablet viewing**:

- Cards should be readable without zooming (minimum 12px text)
- Touch targets minimum 44x44px
- Status colors should be distinguishable on lower-quality screens

**Colorblind-safe status system**:

- Green (#4CAF50) + checkmark icon for "Done"
- Amber (#FF9800) + warning triangle for "At Risk"
- Red (#F44336) + X icon for "Blocked"
- Blue (#2196F3) + play icon for "In Progress"
- Gray (#9E9E9E) + empty circle for "Not Started"
- Purple (#9C27B0) + eye icon for "In Review"

Always pair status color with an icon or text label. Never use color as the sole indicator.

### Contrast Requirements

| Context                                 | Minimum Contrast | Target Contrast |
| --------------------------------------- | ---------------- | --------------- |
| Card title on white background          | 4.5:1 (AA)       | 7:1 (AAA)       |
| Status text on colored background       | 4.5:1 (AA)       | 4.5:1 (AA)      |
| Date/metadata on white                  | 3:1 (AA large)   | 4.5:1 (AA)      |
| Column header text on tinted background | 4.5:1 (AA)       | 7:1 (AAA)       |
| White text on colored card border       | N/A              | N/A             |

## Visual Quality Checklist

- [ ] All cards use consistent dimensions and layout
- [ ] Status colors are applied uniformly (green = done everywhere)
- [ ] WIP limits are visible on column headers
- [ ] Priority indicators use consistent shapes and colors
- [ ] Assignee information is visible at section-level zoom
- [ ] Due dates are displayed in a scannable format
- [ ] The board has a clear "Today" reference point (for timeline views)
- [ ] Column/section headers are large enough to read at full-board zoom
- [ ] Dependencies are shown with appropriate connector styles
- [ ] Overdue items are visually flagged (impossible to miss)
- [ ] The board background is clean and non-distracting
- [ ] Completed items are visually de-emphasized but still accessible
- [ ] The board works at both "quick glance" and "detailed review" zoom levels
- [ ] All status meanings are documented in a legend or use universally understood conventions
