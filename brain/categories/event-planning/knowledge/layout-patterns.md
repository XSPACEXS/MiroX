# Event Planning Boards — Layout Patterns

## Overview

Event boards require layouts that go far beyond grids and columns. The best event boards
create spatial worlds — isometric campuses, organic venue maps, and immersive landscapes
that make stakeholders feel like they are looking down at a real event space. This guide
presents four layout patterns, each suited to a different event board type and visual
ambition level.

---

## Pattern 1: The Isometric Event World

**Best for**: Conference overview boards, event guide boards, premium event showcases
**This is the signature premium pattern**: The one that makes event boards stand out.

The Isometric Event World arranges event zones as pseudo-3D buildings, tents, or spaces
on an isometric grid. Zones are connected by visible pathways. Decorative elements
(clouds, trees, stars, banners) fill the space between zones.

```
+=========================================================================+
|                    [ EVENT TITLE BANNER ]                                |
|              "InnovateTech Summit 2026"                                  |
+=========================================================================+
|                                                                         |
|          *  *                    ___                                     |
|       (cloud)          ___     /   \     *  *                           |
|                       / R \   / MS  \  (cloud)                          |
|   ___                | EG  | |STAGE  |                 ___              |
|  /   \  ~~pathway~~  | IST | |       | ~~pathway~~    /   \            |
| |COMM |              \_____/  \_____/                | SPK  |           |
| |UNITY|    ~~pathway~~     |       ~~pathway~~       |SHOW  |           |
| | HUB |                   |                          |CASE  |           |
|  \___/           +--------+--------+                  \___/            |
|                  |  AGENDA BOARD   |                                    |
|          ___     |  (Schedule at   |      ___                          |
|         /   \    |   a Glance)     |     /   \                         |
|        |COMP |   +--------+--------+    |NETW |                        |
|        |ANION|            |             | OKG  |                        |
|         \___/    ~~pathway~~             \___/                          |
|                          |                              *               |
|              ___         |         ___                  * *             |
|             /   \   ~~pathway~~   /   \               (star)           |
|            |EXPO |              |QUIET|                                 |
|            |HALL |              |ZONE |                                 |
|             \___/                \___/                                  |
|                                                                         |
+=========================================================================+
```

### Key Dimensions

- **Zone shapes**: Rounded rectangles or hexagons, 800-1800px wide depending on importance
  - Main Stage: 1800x1000px (largest zone)
  - Registration: 1200x900px
  - Standard zones: 1000-1200px wide, 700-900px tall
  - Small zones (Quiet, Wellness): 800x600px
- **Pathway width**: 40-80px (main paths wider, secondary paths narrower)
- **Cloud decorations**: 200-400px wide, organic shapes
- **Star decorations**: 30-60px, scattered 5-10 across the board
- **Tree shapes**: 60-100px tall isometric triangles or circles

### Spacing Rules

| Element                               | Spacing                                       |
| ------------------------------------- | --------------------------------------------- |
| Between zones (measured edge-to-edge) | 150-300px (pathways fill this space)          |
| Zone to board edge                    | 120-200px                                     |
| Title banner height                   | 400-600px                                     |
| Decorative element to zone            | 40-80px minimum (never overlap content)       |
| Pathway to zone border                | 0px (pathways connect directly to zone edges) |
| Between decorative elements           | 100-200px (evenly scattered)                  |

### Zone Placement Strategy

Zone placement should follow the attendee journey:

1. **Entry zone** (Registration): Top-left or left-center — the first thing you encounter
2. **Main zone** (Stage/Keynotes): Center of the board — the heart of the event
3. **Social zones** (Community, Networking): Adjacent to the main zone — natural gathering
4. **Activity zones** (Companion, Workshops): Upper and lower edges — satellite experiences
5. **Support zones** (Quiet, Wellness, Expo): Peripheral positions — found when needed
6. **Schedule overview**: Center or bottom-center — the temporal anchor

### Pathway Design

Pathways are the connective tissue of the event world:

- **Main arterial paths**: 60-80px wide, gold/yellow (#FDCB6E or #FFD54F), connecting
  Registration → Main Stage → Networking. These carry the highest traffic.
- **Secondary paths**: 40-50px wide, lighter gold or warm gray, connecting satellite
  zones to the main path.
- **Pathway intersections**: Small circular nodes (100px diameter) where paths cross,
  sometimes marked with a signpost or landmark.
- **Path labels**: Small text along the pathway indicating what the path connects
  ("To Main Stage →" or "← Community Hub").

### Visual Hierarchy Through Zone Design

| Zone Importance                          | Visual Treatment                                                       |
| ---------------------------------------- | ---------------------------------------------------------------------- |
| Primary (Main Stage)                     | Largest frame, boldest border (4px), saturated color, prominent header |
| Secondary (Registration, Speakers, Expo) | Medium frames, standard border (2px), zone color at full saturation    |
| Tertiary (Companion, Networking, Quiet)  | Smaller frames, subtle border (1px), zone color at 70% saturation      |
| Decorative (Clouds, Stars, Trees)        | No border, pastel colors at 30-50% opacity                             |

## Pattern 2: The Venue Floor Plan

**Best for**: Venue layout boards, logistics planning, production staging
**Focus**: Practical spatial accuracy over decorative richness

The Venue Floor Plan uses a top-down architectural view with rooms, corridors, exits,
and service areas clearly delineated. This pattern prioritizes functional accuracy for
production teams.

```
+=========================================================================+
|  VENUE LAYOUT: San Francisco Convention Center — Level 1                |
+=========================================================================+
|                                                                         |
|  +----------+    +---+    +--------------------------+    +----------+  |
|  | ENTRANCE |    | L |    |      MAIN HALL           |    | GREEN    |  |
|  |          |--->| O |    |                          |    | ROOM     |  |
|  | Check-in |    | B |--->| Stage  [====]            |--->| (Spkrs)  |  |
|  | Desks x4 |    | B |    | Seating [800]            |    | 25 cap   |  |
|  |          |    | Y |    | AV Booth [rear]           |    |          |  |
|  +----------+    +---+    +--------------------------+    +----------+  |
|                    |                   |                                 |
|                    |              +----+----+                            |
|                    |              | CORRIDOR|                            |
|                    |              +---------+                            |
|                    |                   |                                 |
|  +----------+    +-+-+    +-----------+-----------+    +----------+     |
|  | BREAKOUT |    |   |    |     EXPO HALL         |    | BREAKOUT |     |
|  | ROOM A   |    | S |    |                       |    | ROOM B   |     |
|  | 60 cap   |    | T |    | Booths: B1-B12        |    | 40 cap   |     |
|  | Projector|    | A |    | Demo stations x6      |    | Projector|     |
|  |          |    | I |    | Lead scanning          |    |          |     |
|  +----------+    | R |    +-----------------------+    +----------+     |
|                  | S |                                                   |
|  +----------+    +-+-+    +-----------+-----------+    +----------+     |
|  | LOUNGE   |         -->| CATERING / DINING     |    | PRAYER   |     |
|  | Networking|            | 400 seated capacity    |    | ROOM     |     |
|  | Round tbls|            | Buffet x3 stations     |    |          |     |
|  +----------+             +-----------------------+    +----------+     |
|                                                                         |
|  [EXIT A]                    [EXIT B]                    [EXIT C]       |
+=========================================================================+
```

### Key Dimensions

- **Room shapes**: Sharp-cornered rectangles (architectural style)
- **Room size**: Proportional to actual room dimensions (if Main Hall is 2x Breakout, frame is 2x)
- **Corridor/pathway**: 60-80px wide, light gray fill
- **Door indicators**: Small breaks (30px wide) in room walls
- **Capacity labels**: Inside each room, bold text

### Spacing Rules

| Element             | Spacing                                         |
| ------------------- | ----------------------------------------------- |
| Between rooms       | 40-60px (corridor width)                        |
| Room wall thickness | 3-4px solid line                                |
| Room label padding  | 20px from top-left corner                       |
| Equipment icons     | 16-24px, positioned where they physically exist |
| Exit markers        | Bottom or sides of the floor plan               |
| Board margins       | 80px                                            |

## Pattern 3: The Agenda Timeline

**Best for**: Agenda boards, schedule visualization, multi-track event planning
**Focus**: Temporal organization with spatial track separation

The Agenda Timeline arranges days horizontally and tracks vertically, with sessions
as colored blocks positioned by time and duration.

```
+=========================================================================+
|  AGENDA: InnovateTech Summit 2026                                       |
+=========================================================================+
|          |      DAY 1 — March 18       |      DAY 2 — March 19          |
|          | 8  9  10  11  12  1  2  3  4| 8  9  10  11  12  1  2  3  4  |
+----------+-----------------------------+-------------------------------+
|          |                             |                               |
| MAIN     | [===KEYNOTE===]  [==PANEL=] | [===KEYNOTE===]  [FIRESIDE]  |
| STAGE    | Dr. Chen         AI Ethics  | M. Rivera        CEO Chat    |
|          |                             |                               |
+----------+-----------------------------+-------------------------------+
|          |                             |                               |
| TRACK A  | [Breakout 1] [Breakout 2]  | [Workshop A]     [Workshop C]|
| (Room A) | UX Research  Data Pipeline | Design Sprint   API Design  |
|          |                             |                               |
+----------+-----------------------------+-------------------------------+
|          |                             |                               |
| TRACK B  | [Breakout 3] [Breakout 4]  | [Workshop B]     [Workshop D]|
| (Room B) | Mobile Dev   ML Ops        | Accessibility   Observability|
|          |                             |                               |
+----------+-----------------------------+-------------------------------+
|          |                             |                               |
| COMMUNITY| [=Lightning Talks=]         |    [=Unconference=]          |
| STAGE    | 5-min rapid fire            |    Self-organized            |
|          |                             |                               |
+----------+-----------------------------+-------------------------------+
|          |     *LUNCH*     *BREAKS*    |     *LUNCH*     *BREAKS*     |
| MEALS    | 12:00-1:00     10:30,3:00  | 12:00-1:00     10:30,3:00   |

|          |              [==Reception==]|           [====Gala Dinner===]|
| SOCIAL   |              6:00-8:00 PM  |           7:00-10:00 PM      |
+----------+-----------------------------+-------------------------------+
```

### Key Dimensions

- **Time column width**: 60-100px per hour
- **Track row height**: 200-300px per track
- **Session blocks**: Width proportional to duration, height matches track row
- **Day separator**: 4px vertical line between days
- **Track label column**: 150-200px wide on the left

### Spacing Rules

| Element                             | Spacing              |
| ----------------------------------- | -------------------- |
| Between track rows                  | 20-30px divider line |
| Between session blocks (same track) | 10-15px (break time) |
| Session block internal padding      | 12-16px              |
| Track label to first session        | 30px                 |
| Day header height                   | 60-80px              |
| Board margins                       | 80px                 |

### Color Coding for Sessions

| Session Type      | Color        | Hex     |
| ----------------- | ------------ | ------- |
| Keynote           | Deep Blue    | #1A237E |
| Workshop          | Pink/Magenta | #AD1457 |
| Breakout/Talk     | Teal         | #00897B |
| Panel             | Purple       | #6A1B9A |
| Lightning Talk    | Orange       | #E65100 |
| Social/Networking | Amber        | #FF8F00 |
| Meal/Break        | Soft Green   | #66BB6A |

## Pattern 4: The Speaker Gallery

**Best for**: Speaker showcase boards, promotional event materials
**Focus**: People-centric presentation with visual impact

The Speaker Gallery arranges speaker cards in a visually engaging grid, with
biographical details, session information, and social links.

### Key Dimensions

- **Keynote speaker card**: 500x600px (large, prominent)
- **Featured speaker card**: 300x400px (medium)
- **Standard speaker card**: 250x300px (compact)
- **Photo area**: Top 50-60% of card
- **Text area**: Bottom 40-50% of card

```
| "Talk Title"       |  14-16pt, Italic, accent color
| Day X, HH:MM      |  12pt, secondary color
| [Room/Stage name]  |  12pt, badge style
+--------------------+
```

## White Space in Event Boards

Event boards have a unique relationship with white space. Unlike other board types where
white space is purely functional, event board "empty space" becomes the event world:

- **Pathway space**: The area between zones is not empty — it is where pathways run
- **Decorative space**: Clouds, trees, and stars occupy the space between zones
- **Breathing room**: Even with decorative elements, 20-30% of the board should be
  visually calm (background color, subtle decoration)

### Space Allocation

| Element Type              | Target Coverage |
| ------------------------- | --------------- |
| Zone frames (event areas) | 45-55%          |
| Pathways and connectors   | 10-15%          |
| Decorative elements       | 10-15%          |
| Open background space     | 20-30%          |

## Zoom Level Design

Event boards must function beautifully at three zoom levels:

### Zoom Level 1: 20-30% (The Event World)

- See the entire event landscape as a world
- Zone names, pathway routes, and decorative atmosphere visible
- Individual session details NOT visible — just the overall structure
- This is the "wow" view that sells the event vision

### Zoom Level 2: 50-70% (Zone Detail)

- Individual zone content is readable
- Speaker names, session titles, and schedule times visible
- Room capacities and logistics visible
- This is the working view for event planning

### Zoom Level 3: 90-120% (Full Detail)

- Speaker bios, AV requirements, catering notes, and setup instructions visible
- Detailed logistics and production notes
- This is the production view for execution teams
