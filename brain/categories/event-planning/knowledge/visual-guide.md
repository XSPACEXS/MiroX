# Event Planning Boards — Visual Guide

## Overview

Event boards are the most visually rich boards in the entire MiRoX system. Unlike product
or design boards where minimal decoration is the rule, event boards embrace visual
richness — isometric worlds, decorative pathways, atmospheric elements, and vibrant color
palettes that capture the energy and excitement of the event itself. This guide provides
the complete visual toolkit for creating event boards that feel like miniature worlds.

## Typography Scale

### Event Board Typography

Event boards use bolder, more expressive typography than other categories because they
serve a promotional function alongside planning.

| Level          | Size    | Weight         | Use Case                                   |
| -------------- | ------- | -------------- | ------------------------------------------ |
| Event Title    | 56-72pt | Bold           | Event name in the title banner             |
| Event Subtitle | 24-32pt | Regular/Italic | Tagline, date, location                    |
| Zone Title     | 32-40pt | Bold           | Zone names ("Main Stage", "Community Hub") |
| Section Header | 24-28pt | Bold           | Sub-sections within zones                  |
| Speaker Name   | 18-22pt | Semi-bold      | Speaker names on cards                     |
| Session Title  | 16-18pt | Semi-bold      | Talk titles, workshop names                |
| Body           | 14-16pt | Regular        | Descriptions, details                      |
| Metadata       | 12-14pt | Regular        | Times, capacities, room numbers            |
| Caption        | 10-12pt | Regular        | Small labels, annotations                  |

### Typography Rules for Event Boards

1. **Event title can be decorative**: Unlike other board types, the event title may
   use a display font or styled treatment. It is the event's brand on the board.
2. **Zone titles should be readable at 30% zoom**: This is critical for the "event world"
   view — zone names are landmarks for navigation.
3. **Speaker names in semi-bold, not regular**: Speakers are people, and their names
   should carry visual weight. Regular weight gets lost on busy event boards.
4. **Use ALL CAPS for zone titles only**: "MAIN STAGE" reads as a landmark.
   "Main Stage" reads as a label. Both are valid — choose based on the event's personality.

### Speaker Card Elements

Speaker cards are the recurring visual unit on event boards:

```
+---Speaker Card---+
| +----+           |
| |    | Speaker   |
| |foto| Name      |
| |    | Title     |
| +----+ Company   |
|                  |
| "Talk Title..."  |
| Day X | HH:MM   |
| [Stage Badge]    |
+------------------+
```

- Photo placeholder: Circle (80-120px diameter) or rounded square
- Name: 18-20pt, bold, primary text color
- Title/Company: 14pt, secondary text color
- Talk title: 14-16pt, italic, accent color
- Time/location: 12pt, metadata style
- Stage badge: Small pill shape with stage color and name

## Contrast and Emphasis

### Creating Focal Points on Event Boards

1. **The Main Stage zone is always the focal point**: Largest zone, most saturated color,
   center position. Everything radiates outward from the main stage.
2. **Pathways guide the eye**: Gold pathways naturally draw attention along their route.
   The eye follows the brightest path from registration to main stage.
3. **Speaker photos create human focal points**: In a board full of shapes and text,
   circular photo placeholders (even as gray circles) draw the eye because the human
   brain is wired to look at faces.
4. **Decorative elements do NOT compete**: Stars, clouds, and trees must be lower contrast
   than content elements. If a cloud is brighter than a zone header, the hierarchy fails.

### The Atmosphere Test

At 30% zoom, the event board should create an emotional response:

- Does it feel like an event? (Not like a spreadsheet or a project plan)
- Can you identify the "heart" of the event? (Main Stage should be obvious)
- Does the color palette feel appropriate for the event type?
- Do the decorative elements create atmosphere without clutter?
- Does it make you want to attend?

## Accessibility

### Event Board Accessibility Requirements

Event boards represent inclusive events, so the boards themselves must be inclusive:

1. **Zone identification**: Every zone must have a text name, not just a color or icon.
   A person with color blindness must be able to identify "Main Stage" from "Expo Hall."
2. **Pathway labels**: Pathways must be labeled with destination text, not just shown
   as colored lines.
3. **High contrast for essential information**: Speaker names, session times, and room
   numbers must meet WCAG AA contrast ratios. Decorative elements may be lower contrast.
4. **Accessible venue information**: Include accessibility annotations on venue layouts —
   wheelchair routes, elevator locations, accessible restrooms, quiet rooms, ASL interpreter
   positions.
5. **Do not rely on isometric view alone**: Include a text-based schedule (agenda board)
   as a companion to the spatial event world for users who process text more easily than
   spatial layouts.

## Visual Consistency Checklist for Event Boards

1. All zone frames use consistent border radius and border weight for their importance tier
2. All pathways use the same color and width for the same tier (main vs secondary)
3. Speaker cards are identically sized and formatted across the entire board
4. Decorative elements are evenly distributed (no clumping in one area)
5. Zone colors are distinct from each other (no two adjacent zones with similar colors)
6. The title banner sets the visual language that the entire board follows
7. Session color coding is consistent between the spatial map and the agenda timeline
8. Text hierarchy is maintained: zone titles > section headers > content > metadata
9. The board works at all three zoom levels (world view, zone view, detail view)
10. Accessibility information is present and clearly marked
