# Event Agenda Board

## Overview

- **Purpose**: Provides a detailed, multi-track event schedule that maps time blocks against parallel sessions, speaker assignments, room locations, and break times. This is the operational heartbeat of any conference — the board that speakers, volunteers, and attendees all reference to know what is happening, where, and when.
- **Best For**: Event coordinators managing multi-track conferences, program managers scheduling parallel sessions, and production teams coordinating room assignments and AV logistics.
- **Complexity**: Medium
- **Board Size**: 4000 x 2500px

## Board Layout

The board is organized as a large time-based grid. The vertical axis represents time (morning to evening), and the horizontal axis represents parallel tracks/rooms. A header row identifies each track. Break times span the full width as horizontal bands. A legend frame sits in the top-right corner.

+--------------------------------------------------------------------+

## Variations

1. **Single-Track Event**: Remove Track B and C columns. Expand Main Stage and Track A to fill the width. Reduce board width to 2500px.
2. **Two-Day Format**: Remove Day 3. Merge unconference and closing keynote into Day 2 afternoon.
3. **Virtual Agenda**: Replace room assignments with platform links (Zoom Room A, Discord Stage, etc.). Add "Recording Available" tags to each session.
4. **Workshop-Heavy Format**: Reduce keynotes to one per day. Expand workshop tracks to 5 parallel rooms. Add prerequisite tags to each workshop.

## Build Instructions

1. Create board at 4000 x 2500px (will grow taller as days are added). Set background to Soft Ivory.
2. Place the title banner (3900 x 300px) at the top with Deep Indigo background.
3. Create track header row with 4 colored rectangles side by side (time column + 4 tracks).
4. For each time slot, create a horizontal row. Use full-width colored bands for breaks/meals. Use column-divided cells for parallel sessions.
5. Place one large sticky note per session containing: talk title, speaker name, time, room, and any notes.
6. Add the Day separator headers between each day with bold text.
7. Build the legend frame in the bottom-right with color swatches and room info.
8. Draw vertical divider lines between tracks and horizontal divider lines between time slots.
9. Add time labels along the left margin in a fixed-width column.
10. Group each day's content into a named Miro frame for easy navigation.

## Tips & Best Practices

- Use Miro tags to mark sessions as "Confirmed", "Tentative", or "Cancelled" so the team sees status at a glance.
- Duplicate this board for each event iteration — archive past agendas as read-only for reference.
- Link each session sticky note to the corresponding speaker card in the Speaker Showcase board.
- During the event, assign a volunteer to update this board in real time with any schedule changes.
- Use Miro's timer widget during sessions to keep speakers on track.
- Print a simplified version (export as PDF, hide sticky note details) for physical signage at the venue entrance.
- Color-blind consideration: add track letter labels (A, B, C) in addition to colors for accessibility.
