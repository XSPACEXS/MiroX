# Build Process — Marketing Enablement Board

## Advanced Build Workflow

Marketing Enablement Boards are the most operationally complex boards to build and maintain. They must be constructed with rigorous structure so that daily updates and team usage do not degrade the board's organization. This guide covers the build process from empty canvas to fully operational war room.

---

## Phase 1: Structure Planning (Before Opening Miro)

**Time: 45-60 minutes**

### Step 1: Inventory Your Needs

- List all marketing activities that need to be tracked (campaigns, content, metrics, assets)
- Identify the team members who will use and update the board
- Determine the key metrics that will appear on the dashboard header
- Identify the cadence for each type of update (daily, weekly, monthly)

### Step 2: Define the Grid Layout

- Determine board dimensions (5000 x 4000px recommended for a full war room)
- Choose number of columns (4 is standard)
- Assign content categories to rows (dashboard > campaigns > pipelines > metrics > resources)
- Calculate frame sizes based on the grid

### Step 3: Define the Color System

- Map status colors to their meanings (green/yellow/red/blue/gray/purple)
- Map channel colors if tracking by channel
- Document the color system — it must be followed consistently

### Step 4: Create a Template

- If this is the first enablement board, plan to save it as a Miro template for future quarters
- Design with reusability in mind — structure should work even as content changes

---

## Phase 2: Board Foundation

**Time: 20-30 minutes**

### Step 5: Create the Board

- Set canvas size: 5000 x 4000px
- Set background: White (#FFFFFF) or Light Gray (#F8F9FA)
- Enable snap-to-grid for consistent alignment

### Step 6: Build the Dashboard Header

- Create a full-width frame (4800 x 300px) at the top
- Set background to Charcoal (#2D3436)
- Add the board title in white (36pt, bold): "FlowBoard Marketing War Room"
- Add date and last-updated timestamp
- Create 4-6 metric cards within the header:
  - Each metric: Value (28pt, bold) + Label (11pt) + Trend (14pt with arrow) + Status dot
  - Metrics: MRR, CAC, Pipeline, MQLs, Active Campaigns, Content Published
- This header should be immediately useful — the "control tower" view

### Step 7: Create the Grid Frames

- Create all section frames based on the grid plan
- Use consistent sizes: Full-width (4800px), Half (2340px), Third (1550px), Quarter (1150px)
- Set consistent gutters (40px between all frames)
- Set all frame borders to 1px, light gray (#DFE6E9)
- Label each frame with a section header (24pt, bold)

---

## Phase 3: Section Construction

**Time: 30-60 minutes per section**

### Step 8: Build Campaign Trackers

For each active campaign:

- Create a campaign card (400 x 350px) within the campaigns row
- Add: Campaign name, type badge (color), status badge, dates, budget, owner
- Add 3-4 KPI rows: Leads, MQLs, Pipeline, CAC with actual vs target
- Add channel breakdown if applicable
- Add key asset links or status indicators

### Step 9: Build Content Pipeline

- Create a Kanban layout within the content section
- 5 columns: Backlog, Brief/Planning, Draft/Creation, Review, Published
- Set column background colors per the specification
- Add WIP limit labels to each column header
- Create 3-5 example content cards to set the format standard
- Each card: Type badge, title, owner, due date, status dot

### Step 10: Build Design/Creative Pipeline

- Similar Kanban layout for design assets
- Columns: Brief, Wireframe, Mockup, Review, Approved, Live
- Include mockup preview areas for key landing pages
- Each mockup card: Page name, version number, status badge, conversion data if live

### Step 11: Build Metrics Dashboard Section

- Create the funnel visualization using rectangles of decreasing width
- Add channel performance cards (one per channel)
- Add trend charts (approximated with rectangles as bar charts)
- Every metric follows the format: Value + Label + Trend + Target + Status

### Step 12: Build Calendar Section

- Create the multi-channel calendar grid
- Rows: Channels (Blog, Email, Social, Paid, Events, PR)
- Columns: Weeks of the current month
- Populate with current month's activities
- Highlight the current week

### Step 13: Build Assets & Resources Section

- Create organized areas for: Brand guidelines, Templates, Approved messaging, Logos, Photography
- Each area has labeled containers for easy asset finding
- Include links to external resources (Figma, Google Drive, etc.) where needed

---

## Phase 4: Connectivity & Workflow

**Time: 15-20 minutes**

### Step 14: Draw Workflow Connections

- Connect campaign cards to their associated content pieces in the pipeline
- Connect metrics to their data sources (labeled connectors)
- Connect calendar entries to their campaign associations
- Use dashed lines for cross-section relationships

### Step 15: Add Status Automation Notes

- Add notes explaining how each status indicator should be updated
- Document which metrics come from which tools
- Create a "How to Update This Board" sticky note in each section owner's area

---

## Phase 5: Operational Setup

**Time: 15-20 minutes**

### Step 16: Assign Section Owners

- Add an "Owner: [Name]" label to every major section
- Add "Updated: [Frequency]" label to every dynamic section
- Create a small "Board Operations" reference card listing all owners and their update responsibilities

### Step 17: Set Up Update Rituals

- Configure any Miro integrations (e.g., Jira, Asana) if applicable
- Set calendar reminders for section owners to update their areas
- Schedule a weekly 15-minute "board walkthrough" in the team meeting

### Step 18: Create the Board Guide

- Add a small "How to Use This Board" frame (optional, linked)
- Contents: Section map, color legend, update cadence, owner directory
- This is for new team members or stakeholders who need orientation

---

## Phase 6: Quality Review & Launch

**Time: 15-20 minutes**

### Step 19: Grid Alignment Check

- Verify all frame tops in the same row share the same Y coordinate
- Verify all frame lefts in the same column share the same X coordinate
- Verify consistent gutters (40px) between all frames
- Verify consistent frame sizes within each grid position

### Step 20: Content Review

- Verify all metrics have: value, label, trend, target, status
- Verify all campaign cards have complete information
- Verify all kanban columns have visible WIP limits
- Verify all status colors are used correctly
- Verify all section owners are labeled

### Step 21: Launch

- Share the board with the marketing team
- Walk through each section in the first team meeting
- Have each section owner demonstrate how they update their area
- Collect feedback after 1 week and adjust structure if needed

## Maintenance Workflow

### Daily (5 minutes per person)

- Move content cards between kanban columns
- Update campaign status indicators
- Add new tasks or content items as they arise

### Weekly (15 minutes, board owner)

- Refresh dashboard metrics
- Update calendar with new week's activities
- Review and archive completed items
- Check for stale sections (not updated in >1 week)

### Monthly (30 minutes, marketing director)

- Add new campaigns for the coming month
- Archive completed campaigns (move to archive area or separate board)
- Update competitive intelligence section
- Review and adjust budget trackers
- Full board walkthrough with team

### Quarterly (2-3 hours, board owner)

- Full board restructure if needed
- Update the grid layout for next quarter's priorities
- Refresh all brand assets and resources
- Archive the current quarter's board as a snapshot
- Build the new quarter's board (often by duplicating and clearing)

---

## Common Build Mistakes

1. **Building without a grid plan**: Leads to inconsistent frame sizes and messy alignment. Always plan the grid first.
2. **Making it pretty before making it functional**: Focus on structure and content first. Visual polish comes last.
3. **Not assigning owners**: Without owners, sections become stale within 2 weeks.
4. **Using too many colors**: Stick to the status color system. Extra colors create confusion.
5. **Forgetting the dashboard header**: Without an at-a-glance summary, stakeholders will not check the board.
