# Visual Guide — Education Boards

## Overview

Education boards serve dual audiences: educators who plan and administrators who review, AND students who learn and navigate. The visual design must balance professional pedagogical structure with approachability and engagement. It should feel rigorous without feeling sterile, warm without feeling juvenile.

---

## Color Palettes

### Palette 1: Academic Classic (The Standard)

The default palette for professional educational contexts — university courses, formal lesson plans, and institutional boards.

| Role       | Color         | Hex     | Usage                                         |
| ---------- | ------------- | ------- | --------------------------------------------- |
| Primary    | Academic Blue | #1565C0 | Headers, objectives, structural elements      |
| Secondary  | Warm Gray     | #6D6D6D | Supporting text, borders, labels              |
| Activity   | Warm Orange   | #EF6C00 | Active learning activities, engagement        |
| Assessment | Deep Purple   | #6A1B9A | Tests, quizzes, evaluation elements           |
| Success    | Sage Green    | #388E3C | Objectives met, outcomes, positive indicators |
| Alert      | Brick Red     | #C62828 | Misconceptions, prerequisites, safety notes   |
| Background | Ivory         | #FFFDE7 | Board background — warm, approachable         |
| Text       | Near Black    | #1A1A1A | Primary reading text                          |
| Subtle     | Light Blue    | #B3D4FC | Card backgrounds, subtle fills                |

**When to use**: K-12 lesson plans, university courses, formal curriculum boards, accreditation documents.

**Why it works**: Academic blue signals professionalism and intellectual authority. Warm orange creates energy for active learning. Deep purple distinguishes assessment from instruction. The ivory background softens the overall feel.

**Activity type color mapping**:

- Blue elements = input/instruction (lecture, reading, demo)
- Orange elements = active learning (discussion, practice, creation)
- Purple elements = assessment (quiz, presentation, exit ticket)
- Green elements = outcomes (objectives, success criteria)

## Typography

### Type Hierarchy

| Level           | Size    | Weight    | Color             | Used For                            |
| --------------- | ------- | --------- | ----------------- | ----------------------------------- |
| Board Title     | 28-32pt | Bold      | White on Primary  | Main header                         |
| Course/Subject  | 18pt    | Regular   | Subtle on Primary | Context info (grade, period, date)  |
| Section Header  | 22pt    | Bold      | Primary           | Frame/section titles                |
| Objective Text  | 14pt    | Semi-bold | Success Green     | Learning objectives                 |
| Activity Title  | 14pt    | Bold      | Activity color    | Activity names                      |
| Activity Detail | 12pt    | Regular   | Dark text         | Activity descriptions, instructions |
| Teacher Note    | 12pt    | Italic    | Secondary gray    | Facilitation notes, tips            |
| Student Voice   | 13pt    | Italic    | Activity color    | Expected responses, quotes          |
| Metadata        | 11pt    | Regular   | Muted gray        | Standards codes, duration, sources  |
| Time Label      | 13pt    | Bold      | Dark              | Minute markers on timeline          |

### Typography Rules

1. **Learning objectives use a distinct visual style** — semi-bold, green, with a checkbox or bullet. They must stand out from activity descriptions.
2. **Teacher notes are italic** — distinguishing facilitation guidance from student-facing content.
3. **Time labels are bold and positioned above activities** — they serve as the "clock" of the lesson.
4. **Standards codes are small and gray** — important for compliance but should not dominate visual hierarchy.
5. **Student voice text is italic in the activity color** — centering anticipated student responses.

## Element Choices

### When to Use Sticky Notes

**Use sticky notes for**:

- Individual learning activities
- Student discussion prompts or anticipated responses
- Quick teacher notes
- Brainstormed lesson ideas (during planning phase)
- Student-generated content (collaborative boards)

**Sticky note color coding**:
| Color | Hex | Use Case |
|-------|-----|----------|
| Sky Blue | #BBDEFB | Input/instruction activities |
| Light Orange | #FFE0B2 | Active learning activities |
| Light Purple | #E1BEE7 | Assessment activities |
| Light Green | #C8E6C9 | Learning objectives and outcomes |
| Yellow | #FFF9C4 | Teacher notes and tips |
| Light Pink | #F8BBD0 | Alerts (misconceptions, prerequisites) |

### When to Use Shapes

**Use shapes for**:

- Lesson plan header bar (rectangle, filled primary)
- Activity timeline segments (rectangles, proportionally sized)
- Objective cards (rounded rectangles with green left border)
- Bloom's taxonomy level indicators (small badge shapes)
- Research question card (large rounded rectangle, prominent)
- Mastery checkpoint markers (circles with check icons)

### When to Use Lines and Connectors

**Alignment connectors**: Thin lines connecting objectives to the activities that address them and the assessments that measure them. Use dashed lines, primary color, 1px.

**Timeline connectors**: Arrows between activity segments showing sequence. Use solid lines, 2px, with arrowheads.

**Concept connections** (study maps): Lines connecting related concepts in the knowledge web. Solid for strong connections, dashed for tangential connections.

## Contrast and Readability

### Special Considerations for Education

Education boards may be:

- Projected on classroom screens (low contrast environment)
- Viewed by students with diverse visual abilities
- Printed in black and white for handouts
- Displayed on tablets and phones

**Design for projection**:

- Minimum text size: 14pt for projected content
- Use high-contrast combinations only
- Avoid light text on light backgrounds
- Test at 50% brightness

**Design for printing**:

- Ensure information is conveyed by shape and label, not just color
- Use borders and patterns in addition to fills
- Include text labels for all color-coded elements

## Decorative Elements

### The Bloom's Level Badge

A small rectangular badge attached to each activity showing its cognitive level:

```
[REMEMBER]  — Light blue, 10pt
[UNDERSTAND] — Medium blue, 10pt
[APPLY]      — Green, 10pt
[ANALYZE]    — Amber, 10pt
[EVALUATE]   — Orange, 10pt
[CREATE]     — Red, 10pt
```

Badge size: 80-100px wide, 22px tall. Attached to top-right corner of activity sticky notes.

### The Timer Bar

A thin horizontal progress bar at the top of each activity column showing cumulative time:

```
[===========.............] 25 of 50 min
```

This gives teachers an instant sense of pacing — are we halfway through the lesson?

### The Differentiation Branch

A small visual fork showing how an activity can be adapted:

```
        [Core Activity]
       /       |        \
  [Scaffold] [Standard] [Extension]
  "Sentence   "Write a   "Write a
   frames      paragraph"  comparative
   provided"              essay"
```

### The Standards Tag

A small, unobtrusive tag linking to curriculum standards:

```
Activity: Label the leaf diagram
[NGSS LS1.C] [CC RST.9-10.7]
```

Tags use a monospace font, small size (10pt), and muted color to stay out of the way while remaining traceable.

## Accessibility Considerations

### For Diverse Learners

- Use sans-serif fonts for readability (especially for dyslexic readers)
- Minimum 12pt for student-facing text
- Pair every color with a text label or shape (for color-blind students)
- Provide alt-text descriptions for any images or icons

### For English Language Learners

- Use simple, clear language in activity descriptions
- Include visual icons alongside text instructions
- Provide vocabulary support (key terms defined on the board)

### For Universal Design for Learning (UDL)

- Multiple means of representation: Show the same information in text, visual, and spatial formats
- Multiple means of engagement: Include varied activity types visible on the board
- Multiple means of expression: Show that assessment accepts different formats
